# ESP32-D0WD 开发板接口速查

来源：`修正ESP32_D0WD_硬件开发手册.md`（板级实测修正版）+ 项目代码验证。
本文只收数字钢琴调试用得到的部分，完整表看原手册。

## 通信与供电

| 项目 | 说明 |
|---|---|
| USB 转串口 | CP2102N，Micro-USB，枚举为 `/dev/ttyACM0` |
| 串口权限 | 用户需在 dialout 组：`sg dialout` 或重新登录 |
| REPL | UART0 = GPIO1(TX)/GPIO3(RX)，被 MicroPython 占用，勿他用 |
| Flash | ZB25VQ32 4MB；GPIO6~11 被 SPI Flash 占用，**不可用** |

## 数字钢琴相关引脚

| GPIO | 板上连接 | 方向 | 关键注意 |
|---|---|---|---|
| 25 | 蜂鸣器驱动（J11→R13→Q3）+ 排针 J5_5/6 | 输出 | DAC1；**高电平→Q3 导通**；断开 J11 可与蜂鸣器隔离 |
| 32 | LED2 绿 + 排针 | 输出 | **低电平点亮**（R10 限流，阳极接 VCC） |
| 33 | LED3 红 + 排针 | 输出 | **低电平点亮** |
| 34 | **KEY2** + 排针 J5_9/10 | **仅输入** | GPIO34~39 无内部上/下拉和输出能力；板载 1k(R9) 上拉，按下=低；`Pin(34, Pin.IN)` 即可，设 PULL_UP 无效 |
| 35 | **KEY1** + 排针 J5_7/8 | **仅输入** | 同上，板载 1k(R12) 上拉，按下=低。注意旧 py 代码注释把 GPIO34 写成 KEY1，属修正前错误命名，以本表为准 |
| 16/17 | I²C SDA/SCL（MPU6050，带外部上拉） | 双向 | 想接 I²S 功放/编解码器时注意避开 |
| 26/27 | MX620B 电机驱动 IN1/IN2 | 输出 | GPIO26 是 DAC2，备用音频通道，但会牵动电机驱动输入 |

## 启动脚约束（上电必须满足，否则不启动/进错模式）

- GPIO12 (MTDI)：启动时必须**低**——外接上拉会导致 flash 电压误判
- GPIO15 (MTDO)：启动时必须**高**
- GPIO0/2：下载模式相关，外接负载要小心

## 音频相关外设能力

| 外设 | 引脚 | 说明 |
|---|---|---|
| RMT | 任意输出脚 | 8 通道，80MHz 基准。本项目用法：`esp32.RMT(0, pin=Pin(25), clock_div=N)`，tick = N/80 µs |
| LEDC PWM | 任意输出脚 | 频率上限高但占空比分辨率随频率下降，做音频 PWM-DAC 不如 RMT 灵活 |
| DAC | GPIO25/26 | 8-bit 0~3.3V。注意：本项目**没有**用 DAC，而是 RMT 输出 PWM 经喇叭机械低通成声 |
| 触摸 | 32/33/27 等 | 与 LED 复用，做扩展琴键时注意冲突 |

## MicroPython v1.18 RMT 语义（实测，易踩坑）

- `rmt.write_pulses(durations, True)`：durations 是**电平持续 tick 数**
  序列，从高电平开始交替（True = 起始电平高）。
- 同通道连续调用：上一块播完才开始下一块，**块间有约 35µs 低电平空隙**
  （与块大小无关）——这就是哒哒音来源。
- `rmt.loop(True)` / `write_pulses` 循环模式：硬件无缝重复，**周期波形
  唯一正确做法**。
- 空闲时写 `rmt.write_pulses((1,), False)` 让引脚回低电平，Q3 截止防
  持续直流。
