## 章节 3.5：第一个智能体任务：绘制 OpenTalon 架构

第一个任务不是生产性的。它是诊断性的。其目的不是产生输出——而是让你在自己的项目上第一次观察感知-推理-行动循环的执行。

在 `opentaion/` 目录中打开 Claude Code。给它这个精确的提示：

```
Do NOT write any code or create any files. Read the project structure
and the CLAUDE.md file, then produce a brief description of what this
project is, what its three components are, and what constraints govern
how it will be built. Ask me if anything is unclear about the project
before writing your description.
```

"不要编写任何代码或创建任何文件"这条指令是有分量的。没有它，Claude Code 可能会将"生成描述"理解为开始构建的邀请。明确的约束将第一个任务保持在探索（Explore）阶段，即探索 → 计划 → 编码 → 提交工作流的范围内。我们还没有在构建。我们是在确认 Claude Code 理解了我们即将要构建什么。

### Claude Code 做了什么

观察工具调用。Claude Code 将使用 Glob 列出目录结构，使用 Read 加载 CLAUDE.md，然后推理它找到的内容：

```
$ claude
> Do NOT write any code or create any files. Read the project structure
  and the CLAUDE.md file...

● Glob: opentaion/**/*
  → cli/, web/, api/ (空目录), CLAUDE.md
● Read: CLAUDE.md
  → 加载了 10 个部分，200+ 行
● 思考中...
```

读取之后，Claude Code 会问一两个澄清问题——或者，如果 CLAUDE.md 足够清晰，就直接进入描述。一个典型的输出：

```
这是 OpenTalon，一个带有 Web 平台的 macOS CLI 编码智能体，
用于用户注册和 Token 用量追踪。

三个组件：
- CLI：Python 智能体（Click + Rich），通过 Homebrew 分发
- API：FastAPI 用量代理（代理到 OpenRouter，计量 Token）
- Web：Vite + React 仪表盘（注册、API 密钥、用量图表）

来自 CLAUDE.md 的约束：
- 技术栈是固定的——不要建议替代方案
- CLI 包管理器：uv（不是 pip）
- 目前还没有代码——下一个里程碑是第 2.5 节

准备好开始时告诉我。
```

这个描述可能与上面的示例有所不同——确切的措辞取决于 Claude Code 如何解释 CLAUDE.md 以及它问什么问题。这种差异是预期的，也不重要。重要的是 Claude Code 已经：

1. 从磁盘读取了实际的 CLAUDE.md（而不是总结了它的描述）
2. 使用 Glob 列出了实际的目录结构（而不是假设那里有什么）
3. 产生了基于它所找到的内容的总结，而不是基于它猜测的内容

### 刚刚发生了什么

那就是感知-推理-行动循环在运行。Glob 和 Read 是感知。"思考中..."步骤是推理。描述是行动。然后循环终止了：响应中没有包含工具调用，所以控制权返回给了你。

这个任务没有产生代码。没有创建任何文件。输出是一段你自己也能写的段落。这些都不重要——教训在于工具调用的序列，而不是输出。

教训在于：Claude Code 现在已经确定了方向。它知道了这个项目是什么、有哪些约束、以及从哪里开始工作。更重要的是，它知道了它读了什么以及它找到了什么——而不是它被告知要假设什么。下一个打开这个项目的会话需要重新读取相同的文件，因为会话记忆不会持久化。但文件本身会持久化。CLAUDE.md 会在那里。目录结构会在那里。而会话前检查清单确保 Claude Code 总是先读取它们。

在 OpenTalon 中，这第一个探索任务是里程碑 2。不是因为输出有什么价值，而是因为完成它确认了开发环境配置正确：Claude Code 可以读取项目、理解 CLAUDE.md、使用 Glob 和 Read 感知结构，并产生接地输出。后续的每一个会话都将以相同的方式开始——由文件引导，而不是由记忆引导。

第一部分到此完成。你已经有了思维模型：智能体是什么、上下文是什么、工具是什么。第二部分构建将这些概念投入实践的平台。
