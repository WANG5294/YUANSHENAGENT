## 章节 3.2：Claude Code 的原生工具

六个工具覆盖了 Claude Code 在开发会话中所做的大部分工作。理解每一个——它的用途、它的成本和它的风险——是良好使用它们的先决条件。

**Read**——将文件内容加载到上下文窗口中。默认从文件开头读取最多 2,000 行。较大的文件可以使用偏移量（offset）和限制（limit）参数分段读取。Read 是用于代码、配置和文档的主要感知工具。它没有副作用：读取文件不会改变它。

**Write**——创建新文件或完全替换现有文件。Write 是生硬的——它替换目标文件的完整内容。对于尚不存在的文件，它是正确的选择。对于已经存在且只需要部分更新的文件，它是一个危险的选择：它会丢弃文件中的所有内容并用新内容替换。对现有文件的 Write 操作会破坏之前存在的任何上下文。

**Edit**——对现有文件进行手术式更改：将特定的字符串或模式替换为新字符串，保持其他所有内容不变。对于现有文件，Edit 比 Write 更受青睐，因为它精确显示更改了什么、保留了未更改的所有内容，并且如果找不到目标字符串会安全地失败。当 Claude Code 编辑一个你想要审查的文件时，你审查的是差异（diff），而不是替换。当文件很长时，这种区别极其重要。

**Bash**——在持久的 Shell 会话中执行 Shell 命令。工作目录在会话内的 Bash 调用之间持久化；其他 Shell 状态（在子 Shell 中设置的环境变量、别名）则不持久化。Bash 是最强大的工具，也是最危险的：它可以运行测试、编译代码、安装包、Git 提交、删除文件和调用外部 API。使 `pytest` 有用的同一个工具也使 `rm -rf` 成为可能。第 8 章是关于如何将 Bash 限制为你意图的范围。

**Glob**——通过模式搜索文件。`**/*.py` 查找项目树中的每一个 Python 文件。Glob 结果按修改时间排序，用于发现仓库中存在的文件而不读取任何文件的内容。它快速、廉价且安全。

**Grep**——通过正则表达式在一个或多个文件中搜索文件内容。`agent.run_loop` 查找包含该函数调用的每一个文件。Grep 是大型代码库的导航工具——一个有针对性的 Grep 结果通常只有 100 个 Token，而它指向的完整文件可能有 5,000 个。在需要找到相关代码时，先使用 Grep 再使用 Read，而不是读取所有代码。

风险层级从 Read（无副作用）经过 Edit（修改文件但保留上下文）和 Write（完全替换文件）到 Bash（执行任意命令）排列。一个设计良好的智能体任务会刻意地按这个层级推进：先观察（Read、Glob、Grep），然后谨慎地行动（Edit），最后只执行前序观察所证明必要的操作（Bash）。

另外两个工具将智能体的感知范围扩展到本地文件系统之外。**WebFetch** 检索 URL 的内容并根据提示进行总结。**WebSearch** 执行搜索查询并返回结果。这些是允许 Claude Code 读取文档、查看库的更新日志、或在本地代码库中没有的问题答案时进行查找的工具。

**TodoWrite** 与其他工具不同：它不与文件系统或网络交互。它在智能体的上下文中维护一个结构化的任务列表——带有状态字段（pending、in-progress、completed）的项目，Claude Code 可以在工作时读取和更新。对于有许多步骤的长时间任务，TodoWrite 是智能体跟踪自己位置而不需要重新读取整个对话历史的机制。

在 OpenTalon 中，这些工具是 Claude Code 用来构建 OpenTalon 本身的工具——而且它们也是 OpenTalon 将通过 CLI 向其用户暴露的同一个工具集。当我们在第 9 章实现 CLI 智能体循环时，Read、Edit 和 Bash 将成为 OpenTalon 自己工具集的核心。我们不仅是在抽象地学习这些工具。我们是在学习它们，因为我们将自己实现它们的对应版本。

这些工具存在于一个反馈循环之内。循环的价值不是来自任何单次工具调用，而是来自调用之间发生的事情——将一个观察结果转化为下一个行动规范的推理步骤。下一节直接考察这种反馈结构。
