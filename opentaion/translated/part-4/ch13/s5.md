## 第 13.5 节：为 OpenTalon 安装 BMAD V6

前四节解释了原则。本节进行安装。BMAD V6 通过一个命令安装，但在安装过程中做出的配置选择决定了哪些工作流程可用。现在正确配置意味着后续章节可以继续进行，而无需重新审视设置。

### 安装命令

```bash
cd opentaion/
npx bmad-method install
```

BMAD 的安装程序以交互方式运行。它呈现一系列提示，询问包含哪些模块。对于 OpenTalon：

```
? 选择要安装的模块
  ◉ BMM（核心——所有规划与开发工作流程）
  ◉ TEA（测试卓越加速器，Testing Excellence Accelerator）
  ○ DevOps（CI/CD 集成工作流程）——暂不安装
  ○ Data Science（机器学习/分析工作流程）——不相关

? 安装位置
  ◉ 项目根目录（推荐用于单项目仓库）
  ○ Monorepo 根目录（跨所有包共享安装）

? 安装 Claude Code 集成？
  ◉ 是——安装 .claude/commands/ 集成
  ○ 否
```

选择 BMM 和 TEA。跳过 DevOps（第 17 章将直接配置 CI/CD，不通过 BMAD）。跳过 Data Science。安装在项目根目录。安装 Claude Code 集成。

`.claude/commands/` 目录是 Claude Code 查找自定义斜杠命令的地方——机制与第 5 章中的技能系统相同，但 BMAD 在安装期间用自己的命令填充它。

### 验证安装

```bash
# 在 Claude Code 会话中：
/bmad-help
```

预期输出：

```

可用智能体：
  /bmad-agent-bmm-analyst      Mary — 业务分析师
  /bmad-agent-bmm-pm           John — 产品经理

  /bmad-agent-bmm-qa           Quinn — QA
```

### 更新根 CLAUDE.md

BMAD 安装了自己的 CLAUDE.md 更新指南，但它是通用的。向 OpenTalon 根 CLAUDE.md 添加一个项目特定的 BMAD 节，解释何时使用 BMAD 智能体与直接实现：

```markdown
## BMAD 工作流程

本项目使用 BMAD V6 进行 Web 平台的规划。

**使用 BMAD 智能体进行：**
- 创建或修改 PRD、架构或 UX 规格
- 从规划文档生成故事文件
- 任何需要产生 _bmad/artifacts/ 文档的规划工作

**使用直接实现进行：**
- cli/SPEC.md 中指定的 CLI 功能
- 范围明确的错误修复
- 单个函数的添加
- 任何 SPEC.md 或故事文件提供完整规格的任务

**不要使用 BMAD 智能体进行直接实现。** Mary、John、Sally、
Winston、Bob 和 Quinn 是规划智能体。Amelia 是实现
智能体——仅通过 _bmad/stories/ 中的故事文件调用她。
```

最后一条规则防止了一个常见错误：要求 Mary 或 Winston 实现代码。它们的角色和系统提示是为规划设计的，而非实现。将它们用于实现会产生规划质量的输出，而需要的是实现质量的输出。

### 在 OpenTalon 中

在 OpenTalon 中，当 BMAD V6 安装完成、`/bmad-help` 返回预期输出且 CLAUDE.md BMAD 节已提交时，达到里程碑 M12。`_bmad/artifacts/` 目录此时为空——这是正确的。它将在第 14 章和第 15 章中随着规划文档的产生而填充。第 14 章以 Mary 为 OpenTalon Web 平台进行的引导式需求获取会话开始。

### 刚刚发生了什么

BMAD V6 已安装，Claude Code 知道它的存在。七个智能体角色可作为斜杠命令使用。`_bmad/` 目录结构已准备好接收规划文档。根 CLAUDE.md 告诉 Claude Code 何时使用 BMAD 智能体以及何时直接实现。

第四部分的其余三章将实践此安装：第 14 章运行规划智能体完成完整规划序列，第 15 章产生架构，第 16 章将计划转换为已实现的软件。
