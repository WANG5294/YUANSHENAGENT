## 第 8.2 节：规则语法与拒绝优先层级

权限模式控制默认行为。规则控制例外情况。这两个系统协同工作：模式设定基线，规则在此基础上叠加具体的允许或拒绝决策。理解规则如何组合——特别是拒绝优先原则——对于构建一个安全配置至关重要，这个配置既不会意外地将 Claude Code 锁在合法操作之外。

### 允许和拒绝规则

规则在 `settings.json` 的 `permissions` 部分中以字符串形式指定。每条规则指定一个工具名称，可选地带有过滤器，并标记为 `allow` 或 `deny`：

```json
{
  "permissions": {
    "allow": [
      "Bash(git log:*)",
      "Bash(git diff:*)",
      "Bash(uv run pytest:*)"
    ],
    "deny": [
      "Bash(rm -rf:*)",
      "Bash(curl:*)",
      "WebFetch"
    ]
  }
}
```

**规则语法：**

- `ToolName` —— 匹配该工具的所有使用。`Bash` 允许或拒绝所有 Bash 调用。
- `ToolName(pattern)` —— 匹配输入包含该模式的工具调用。`Bash(git log:*)` 匹配命令以 `git log` 开头的 Bash 调用。`*` 是通配符。
- `mcp__server__tool` —— 匹配特定的 MCP 工具。`mcp__postgres__execute_query` 只匹配 postgres 服务器上的 execute_query 工具。
- `mcp__server__*` —— 匹配来自特定服务器的所有工具。`mcp__github__*` 允许或拒绝所有 GitHub MCP 工具。

**过滤器匹配的内容：** 对于 Bash，过滤器针对命令字符串进行匹配。对于 Read 和 Write，它针对文件路径进行匹配。对于 MCP 工具，它针对工具名称进行匹配。

### 拒绝优先层级

规则评估遵循固定的优先级：任何级别的拒绝都优先于任何级别的允许。

更具体地说，从高到低的优先级是：

1. **托管设置（Managed settings）**（组织策略）——由管理员为企业账户配置；个体开发者无法覆盖这些设置
2. **CLI 参数** —— 启动 Claude Code 时传递的标志（`--allow-file-write`）
3. **本地项目设置（Local project settings）**（`.claude/settings.local.json`）——不提交到 git，用户特定的覆盖
4. **共享项目设置（Shared project settings）**（`.claude/settings.json`）——提交到 git，在团队中共享
5. **用户设置（User settings）**（`~/.claude/settings.json`）——个人默认值

第 4 级（共享项目设置）的拒绝规则不能被第 5 级（用户设置）的允许规则覆盖。无论拒绝在层级中相对于允许处于什么位置，拒绝都优先。这防止了个别用户的设置削弱项目为所有人建立的安全约束。

实际影响：如果你在 `.claude/settings.json` 中放置一条拒绝规则并提交它，该规则适用于打开此项目的每个会话，包括具有提升用户级权限的会话。这是项目级安全约束的正确位置。

### 一个实际配置

对于 OpenTalon 开发，`.claude/settings.json` 中的权限部分应反映项目的实际需求：

```json
{
  "permissions": {
    "allow": [
      "Bash(uv run:*)",
      "Bash(git:*)",
      "Bash(npm:*)",
      "Bash(npx:*)",
      "mcp__github__*",
      "mcp__playwright__*"
    ],
    "deny": [
      "Bash(rm -rf:*)",
      "Bash(curl * | bash:*)",
      "Bash(wget * | bash:*)"
    ]
  }
}
```

该配置允许所有 `uv run` 命令（测试、lint、代理）、所有 git 操作、npm 操作、GitHub 和 Playwright MCP 工具，并拒绝危险的特定 Bash 模式：`rm -rf` 以及从网络执行不受信任代码的管道到 bash 模式。

`mcp__postgres__*` 工具不在允许列表中——当数据库配置好且正确的只读连接字符串可用时，它们将被添加。

### 与 OpenTalon 的关联

在 OpenTalon 中，项目设置文件是 `.claude/settings.json`，与 CLAUDE.md 文件和钩子脚本一起提交。那里的权限部分是项目永久的、版本控制的安全配置。当新会话打开 OpenTalon 时，它会自动继承这些规则——无需手动设置，无需依赖记忆。接下来的两节将为这个配置添加更多层次：操作系统级沙箱和一个识别规则正在防范什么的威胁模型。
