## 第 7.3 节：PostgreSQL MCP：Claude Code 查询数据库

GitHub MCP 服务器让 Claude Code 能够访问代码审查层。PostgreSQL MCP 服务器让它能够访问数据层。当生产环境中出现使用追踪 bug——用户令牌计数错误、账单条目缺失、时间序列出现异常尖峰——第一步诊断就是查询数据库。没有 PostgreSQL MCP，这一步需要开发者编写查询、运行它，并将结果粘贴回来。有了它，Claude Code 直接编写和执行查询。

这不是一个微不足道的便利。数据库调试中的反馈循环是：假设 → 查询 → 结果 → 修正假设。当 Claude Code 自己闭合循环而不是等待开发者执行每个查询并报告结果时，每个周期都更快。

### PostgreSQL MCP 提供的能力

PostgreSQL MCP 服务器提供三类工具：

**Schema 检查** —— 描述表、列出列及其类型、显示约束和索引、映射外键关系。当 Claude Code 遇到不熟悉的表结构时，它直接进行检查，而不是靠记忆推理。

**SELECT 查询** —— 执行任意 SELECT 查询并返回结果。MCP 服务器使用连接字符串进行配置；Claude Code 发送查询，服务器执行它，并将结果行作为结构化数据返回。

**查询解释** —— 在查询上运行 EXPLAIN ANALYZE 以查看执行计划并识别性能问题。

该服务器不支持 INSERT、UPDATE、DELETE 或 DDL 操作。这是一个刻意的配置选择，而不是协议的限制。在开发过程中，Claude Code 绝不应该通过 MCP 服务器修改生产数据库。Schema 变更通过 Alembic 迁移进行。数据修改通过应用程序代码进行。MCP 服务器在设计上是只读的。

### 使用 Supabase 进行配置

Supabase 为每个项目提供一个 PostgreSQL 连接字符串。MCP 服务器直接连接到此数据库：

```bash
# Install the PostgreSQL MCP server
npm install -g @modelcontextprotocol/server-postgres

# Add as a project-level MCP server (connection string goes in .env, not committed)
claude mcp add postgres \
  --scope project \
  --transport stdio \
  --command "npx @modelcontextprotocol/server-postgres" \
  --env DATABASE_URL="postgresql://postgres:***@db.[project-ref].supabase.co:5432/postgres"
```

连接字符串包含凭据——不要将其提交到 `.mcp.json`。将其添加到 `.env` 并引用环境变量：

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "${DATABASE_URL}"
      },
      "type": "stdio"
    }
  }
}
```

`${DATABASE_URL}` 语法从会话的环境中读取，反过来从 `.env` 中读取。凭据永远不会出现在已提交的文件中。

### 一个具体的调试示例

在第 16 章，当使用追踪端点上线后，像这样的场景变得可操作：

```
> 一位用户报告说，他们在仪表板中昨天显示的使用量为 0，
> 但他们运行了几个会话。找出原因。
```

有了 PostgreSQL MCP 连接，Claude Code 将：
1. 查询 `usage_logs` 获取用户昨天的记录
2. 检查记录是否存在（它们可能存在于不同的时间戳）
3. 查询仪表板聚合视图以了解其聚合方式
4. 识别不匹配情况（时区问题、聚合窗口偏移等）
5. 报告根本原因，并附上特定查询结果作为证据

没有 PostgreSQL MCP，步骤 1 到 4 每一步都需要开发者进行上下文切换。有了它，调查在一个 Claude Code 任务中就能完成。

### 与 OpenTalon 的关联

在 OpenTalon 中，PostgreSQL MCP 服务器连接到支撑使用追踪平台的 Supabase 数据库。在数据库存在之前（现在，第 7 章时），这个服务器无法连接——还没有数据库可以连接。在 `.mcp.json` 中使用 `${DATABASE_URL}` 占位符配置条目，以便在第 15 章数据库配置好时准备就绪。一旦 `DATABASE_URL` 在 `.env` 中设置好，MCP 连接本身就会激活。现在构建连接基础设施意味着在紧张的第四部分构建期间减少一个设置步骤。
