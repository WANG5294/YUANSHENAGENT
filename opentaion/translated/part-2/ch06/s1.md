## 第 6.1 节：12 个生命周期事件

技能（Skills）规定了 Claude Code 在显式调用命令时的行为。钩子（Hooks）则规定了 Claude Code 在代理循环的特定节点自动执行的行为，无需任何显式调用。它们是技能层之下的自动化层——负责强制执行约束和运行质量检查，无需人类记得去要求它们。

Claude Code 暴露了十二个生命周期事件，每个事件对应执行周期中的一个特定时刻。理解每个事件的触发时机，是知道哪些钩子该放在哪里的前提。

### 十二个事件

**会话事件（Session events）** 在会话本身开始或结束时触发：

- `SessionStart` —— 在 Claude Code 会话打开时触发。用于设置环境变量、检查所需工具是否可用，或加载会话所需的任何外部上下文。
- `SessionEnd` —— 在会话关闭时触发。实践中很少使用；会话可能突然结束（终端关闭、超时），该事件并非在所有情况下都能可靠触发。
- `Setup` —— 在处理第一条用户消息之前触发。与 SessionStart 类似，但触发时间稍晚，在 CLAUDE.md 文件加载完成之后。

**工具事件（Tool events）** 在单个工具调用前后触发：

- `PreToolUse` —— 在工具调用执行之前触发。钩子会接收工具名称和输入，并可以批准、阻止或修改该调用。这是强制执行点：如果 PreToolUse 钩子阻止了一个调用，该工具就不会执行。
- `PostToolUse` —— 在工具调用成功完成后触发。接收工具名称、输入和输出。用于自动化质量检查：在文件写入后运行 lint，在 Bash 命令后更新日志。
- `PostToolUseFailure` —— 在工具调用失败时触发。接收错误信息。用于错误日志记录或告警。

**代理事件（Agent events）** 在子代理执行时触发（多代理工作流）：

- `SubagentStart` —— 在生成的子代理开始时触发。
- `SubagentStop` —— 在子代理完成或被终止时触发。

**用户事件（User events）** 在用户交互时触发：

- `UserPromptSubmit` —— 在用户提交消息时触发，在 Claude Code 处理之前。可用于向每条消息注入额外的上下文。
- `PermissionRequest` —— 在 Claude Code 请求用户授权执行某个操作时触发。钩子可以自动批准或自动拒绝特定类别的请求。

**完成事件（Completion events）** 在工作完成或代理需要关注时触发：

- `Stop` —— 在代理循环终止时触发（任务完成或进入错误状态）。
- `Notification` —— 在 Claude Code 想提醒用户时触发——通常是在长时间运行的任务完成且代理等待输入时。

### 实际常用的子集

十二个事件是完整的列表。在实践中，四个事件覆盖了绝大多数实际用例：

**`PreToolUse`** —— 强制执行钩子。在危险操作执行之前拦截。检测代码中 API 密钥的钩子放在这里。阻止特定目录外 `rm -rf` 的钩子放在这里。任何你希望是结构性而非建议性的约束，都应该放在 PreToolUse 钩子中。

**`PostToolUse`** —— 质量自动化钩子。每写一个 Python 文件后，运行 black 和 ruff。每次测试运行后，检查结果并记录日志。任何希望在操作后自动执行的质量检查都应放在这里。

**`Notification`** —— 开发者生产力钩子。当 Claude Code 完成一个长任务并等待输入时，触发一个操作系统级通知，这样你可以关闭终端窗口切换到其他应用。这个钩子将 Claude Code 从一个需要你盯着看的终端，变成了一个后台工作者。

**`SessionStart`** —— 环境设置钩子。检查所需 CLI 工具是否已安装，验证环境变量是否已设置，运行任何应在第一个任务之前执行的初始化工作。

### 钩子的配置方式

钩子在 `.claude/settings.json`（项目级，通过 git 共享）或 `~/.claude/settings.json`（用户级，个人使用）中配置。结构将生命周期事件映射到钩子数组，每个钩子指定一个匹配器（matcher）和一个动作（action）：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/format-python.sh"
          }
        ]
      }
    ]
  }
}
```

`matcher` 字段指定哪些工具调用会触发该钩子。`type` 字段指定钩子的动作。最常见的类型是 `command` —— 一个 shell 脚本，通过 stdin 接收工具调用的上下文，并可以返回决策或输出。

安全提示：钩子脚本在会话启动时被快照。如果你在会话中途修改了钩子脚本，修改不会生效，直到会话重新启动。如果行为与预期不符，使用 `/hooks` 查看当前活动的钩子。

### 与 OpenTalon 的关联

在 OpenTalon 中，钩子系统是自动化质量基础设施的最后一块拼图。第 5 章赋予了 Claude Code 搭建代码框架的技能。第 6 章赋予它自动强制执行代码质量的约束。当我们在第 6.3 节和第 6.5 节编写保存时格式化钩子和 API 密钥检测钩子时，项目中的每次文件写入都将自动进行格式化和秘密扫描——无需任何显式提醒，无需任何手动步骤。开发者的工作变成了指定意图；钩子负责处理正确性。
