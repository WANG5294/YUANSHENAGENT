## 第 6.4 节：通知钩子：操作系统级提醒

前面两节增加了代理循环内部的自动化：对 Claude Code 能做什么的约束，以及对其已完成工作的质量检查。通知钩子的性质不同。它不影响代码或工具调用。它影响的是开发者的工作流程：具体来说，它消除了盯着终端看的需要。

长时间的代理任务需要时间。从代码库扫描中构建规范文档、根据测试套件实现完整的 CLI 命令、调试多文件问题——这些会运行五到十五分钟。一个开发者在这段时间里盯着终端，并没有在做任何有用的事情。但如果他们切换到其他应用，错过了 Claude Code 完成并请求输入的时刻，会话就会闲置等待。

Notification 钩子就是解决方案。它在 Claude Code 想要提醒用户时触发——代理正在等待输入、一个长任务已完成、一个错误需要关注。钩子触发一个操作系统级的 macOS 通知。开发者的屏幕上出现一个通知，他们切换回终端，会话继续。

### macOS 实现

macOS 自带 `osascript`，AppleScript 执行器。无需第三方工具，无需额外安装。以下一行命令触发一个原生的 macOS 通知：

```bash
osascript -e 'display notification "Claude Code needs your attention" with title "Claude Code" sound name "Glass"'
```

完整的通知钩子：

```bash
#!/bin/bash
# .claude/hooks/notify-macos.sh
# Fires on the Notification lifecycle event

INPUT=$(cat)
MESSAGE=$(echo "$INPUT" | python3 -c "
import json, sys
d = json.load(sys.stdin)
print(d.get('message', 'Claude Code needs your attention'))
" 2>/dev/null || echo "Claude Code needs your attention")

osascript -e "display notification \"$MESSAGE\" with title \"Claude Code\" sound name \"Glass\""
```

settings.json 配置项：

```json
{
  "hooks": {
    "Notification": [
      {
        "matcher": ".*",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/notify-macos.sh"
          }
        ]
      }
    ]
  }
}
```

`.*` 匹配器会触发所有 Notification 事件。通知中包含 Claude Code 传递给 Notification 事件的消息——通常是对它需要什么（"正在等待用户确认以删除文件"）或完成了什么（"测试套件通过。准备下一步"）的描述。

### 工作流程的变化

实际的影响是显著的。没有通知钩子时，一个 10 分钟的任务要么需要盯着终端看，要么需要定期回来检查。有了钩子，工作流程变成：启动任务，将 Claude Code 窗口最小化到后台终端，做其他工作，收到 macOS 通知，切换回来。终端从焦点变成了背景。

对于 OpenTalon 的构建——其中包括几个长的合成任务，如编写 SPEC.md、运行完整的测试套件以及构建完整的 Web 仪表板——这个钩子将十五分钟的等待时间从阻塞间隔变为后台操作。

`sound name "Glass"` 在通知触发时播放 macOS 的 Glass 声音。将其更改为任何 macOS 系统声音名称（Basso、Blow、Bottle、Frog、Funk、Hero、Morse、Ping、Pop、Purr、Sosumi、Submarine、Tink），或者完全移除 `sound name` 子句以获得静默通知。

### 扩展该模式

同样的通知钩子可以扩展到其他通信渠道。在团队环境中工作的开发者可以将通知路由到 Slack webhook，在一个长批次任务完成时向频道发送消息。钩子结构支持 HTTP 类型的钩子，可以向 URL 发送 POST 请求——将 `command` 类型替换为 `http` 并提供端点。相同的 Notification 生命周期事件，不同的传递机制。

对于 OpenTalon 的单人开发，macOS 通知已经足够。没有团队频道，反馈周期足够紧凑，操作系统级通知覆盖了所有场景。关于 CI/CD 的章节（第 17 章）将此模式扩展到 GitHub Actions 通知，用于生产流水线事件。

### 与 OpenTalon 的关联

在 OpenTalon 中，通知钩子在 BMAD 辅助阶段（第四部分）尤为有价值，那时一个单一任务——"分析 PRD 并编写架构文档"——可能运行十五到二十分钟。没有通知时，第四部分的会话需要持续的终端关注。有了钩子，开发者可以在架构合成运行时做其他事情，macOS 通知在架构文档准备好供审阅时发出信号。该钩子从第 6 章开始就位，因此本书中的每个长时间运行的会话都受益于它。
