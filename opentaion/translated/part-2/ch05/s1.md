## 章节 5.1：内置命令：完整参考

Claude Code 自带一组斜杠命令（slash commands），从第一次按键即可使用。无需配置，无需设置。它们作用于运行中的会话、模型、工作流和项目配置。其中一些在 Tab 补全列表中可见；少数不那么显眼但对日常开发很重要。本节是参考：每个命令做什么以及何时使用它。

在列表开始之前有一个澄清：这些与技能不同。内置命令是 Claude Code 本身的一部分——无论你在哪个项目中、无论你有什么 `.claude/` 目录，它们都存在。技能（在第 5.2 节中介绍）是你自己编写的项目特定命令。内置命令不能被定制；技能可以。

### 会话管理

**`/clear`** — 清空对话历史并重新开始。此命令后上下文窗口为空。CLAUDE.md 文件将在下一条消息时重新加载。在不相关的任务之间使用，或者当你想开始一个新任务而不携带前一个任务累积的上下文时使用。代价是丢失会话中已经建立的一切——只有在代价可接受时使用 `/clear`。

**`/compact`** — 将对话历史压缩为摘要，释放上下文空间而不完全丢失会话的工作。Claude Code 会写入已经完成的工作摘要，然后从该摘要继续。当上下文在中途填满且你需要继续同一个任务而不从头开始时使用。摘要可能会丢失一些细节——重要的上下文应该放在文件中，而不仅仅是对话中。

**`/cost`** — 显示当前会话的 Token 数和预估成本。对于校准你对特定类型任务的成本感知很有用。当看到一次大型调试会话消耗了 40K Token 后，未来同类会话的成本就变得可预测了。

**`/status`** — 显示当前模型、思考模式、权限设置和其他一些会话参数。用以确认你所期望运行的内容确实在运行，特别是在切换模型或调整权限之后。

### 模型和推理控制

**`/model`** — 在会话中切换活动模型。示例：`/model claude-opus-4-6` 切换到 Opus 处理复杂的架构决策，然后切换回 Sonnet 进行实现。会话中切换模型会保留对话上下文——模型更改在下一消息时生效。

**`/effort`** — 调整思考 Token 预算。这将在第 5.5 节中详细讨论。简而言之：它控制 Claude Code 在回复之前进行多少内部推理。`/effort low` 快速且便宜；`/effort max` 彻底但昂贵。第 5.5 节涵盖了在 OpenTalon 开发中每个级别何时适用。

### 工作流命令

**`/review`** — 触发对当前更改的代码审查。Claude Code 读取差异，应用 CLAUDE.md 中的项目编码约定，并生成带有具体改进建议的结构化审查。在提交涉及多个文件的更改之前使用。

**`/pr_comments`** — 读取当前 Pull Request 上的公开评论（如果你在 GitHub 连接的工作流中）并处理它们。当 PR 有多个审阅者评论并且你希望 Claude Code 按顺序处理所有评论时很有用。

**`/release-notes`** — 从自上个标签以来的提交历史生成发布说明。它将提交总结为用户可读的语言，按类型（功能、修复、改进）分组，并生成你可以编辑的草稿。

### 配置和内省

**`/memory`** — 打开 CLAUDE.md 层级进行编辑。显示每个级别（组织、用户、项目、子目录）的活动记忆文件，并允许你添加或编辑条目。当你想在会话中临时添加规则时使用，而不是手动导航到 `~/.claude/CLAUDE.md` 或项目 CLAUDE.md。

**`/permissions`** — 显示当前权限设置并允许你调整。权限模式（自动批准、每次使用工具时询问等）在第 8 章中介绍。当你想临时提升或限制 Claude Code 的权限而不重新启动会话时使用 `/permissions`。

**`/mcp`** — 列出连接的 MCP 服务器及其可用工具。用于验证你的 GitHub、PostgreSQL 或 Playwright MCP 服务器是否已连接并暴露出你期望的工具。第 7 章构建完整的 OpenTalon MCP 栈。

**`/hooks`** — 列出活动的生命周期钩子。显示哪些事件注册了钩子、它们运行什么命令以及它们是阻塞的还是非阻塞的。用于诊断钩子为什么触发了或没触发。第 6 章涵盖完整的钩子系统。

### 实用工具

**`/doctor`** — 对 Claude Code 安装运行诊断检查：检查所需二进制文件、验证 `.claude/` 目录结构、测试 MCP 服务器连接、报告任何问题。当某些东西没有按预期工作且你无法确定原因时运行。

**`/init`** — 扫描当前仓库并根据发现的内容生成一个起步 CLAUDE.md：package.json 文件识别前端栈，pyproject.toml 识别 Python 栈，现有配置文件提示 linting 和测试约定。结果是一个需要编辑的 CLAUDE.md 草稿，而不是一个完成文件——但对于一个开始没有 CLAUDE.md 的现有项目来说，这比从头写要快。对于 OpenTalon，我们在任何代码存在之前就编写了 CLAUDE.md，所以 `/init` 在这里没用。对于棕地项目，它是一个起点。

**`/vim`** — 在 Claude Code 输入中切换 Vim 键绑定。个人偏好；没有功能影响。

**`/terminal-setup`** — 配置终端集成：设置 Shell 补全、配置在终端状态行中显示 Claude Code 状态的提示集成，并启用其他终端级功能。在新机器上运行一次。

### 对 OpenTalon 最重要的三个命令

在 OpenTalon 开发中，有三个命令将在剩余章节中反复出现：

- **`/compact`** —— 当长时间构建会话填满上下文且你需要继续而不从头开始时使用
- **`/clear`** —— 在主要里程碑之间使用，以开始每个新阶段而不带上个阶段的上下文噪音
- **`/effort`** —— 用于为每种任务类型调整推理深度：脚手架用低，实现用中，调试用高，架构用最高
