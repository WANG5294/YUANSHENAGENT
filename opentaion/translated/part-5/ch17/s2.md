## 17.2 节：GitHub Action：自动 PR 审查

人类在审查拉取请求时，拥有代码审查工具所不具备的上下文：CLAUDE.md 中的约定、架构决策、之前会话中确立的模式。如果给予相同的文件，自动化 Claude Code 审查工具也拥有同样的上下文。在每次 OpenTalon 拉取请求上运行的 GitHub Action 恰好提供了这一点。

### Action

Anthropic 提供了 `anthropics/claude-code-action@v1` —— 一个官方的 GitHub Action，在 CI 环境中运行 Claude Code。

```yaml
# .github/workflows/pr-review.yml
name: Automated PR Review

on:
  pull_request:
    types: [opened, synchronize]

permissions:
  contents: read
  pull-requests: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Need full history for diff

      - name: Run Claude Code review
        uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          max_turns: 5
          allowed_tools: "Bash,Read,Glob,Grep"
          direct_prompt: |
            Review this pull request.

            Changed files: ${{ steps.changed-files.outputs.all_changed_files }}

            Check for:
            1. Violations of conventions in CLAUDE.md
            2. Missing tests for new functionality
            3. Security issues (exposed credentials, SQL injection,
               missing input validation)
            4. API contract violations against architecture.md

            Format your review as:
            CRITICAL: [list any critical issues that must be fixed]
            WARNING: [list non-critical issues worth addressing]
            OK: [confirm what looks good]

            If no CRITICAL or WARNING issues, output only "OK: No issues found."
```

### 审查的职责

Action 在与 CI 环境相同的仓库上下文中运行。Claude Code 读取 CLAUDE.md，读取变更文件，并生成结构化的审查评论，发布到拉取请求上。

审查不能替代人工审查。它是第一道关卡，用于捕获：
- 明显的约定违反（导入了技术栈中未包含的库、跳过了 CLAUDE.md 中的规则）
- 缺少测试（新函数没有对应的测试文件）
- 可通过模式分析检测的安全问题（代码中的 API 密钥、未检查的用户输入传入 SQL）
- API 合约违反（响应格式与 `architecture.md` 不符）

人工审查员从自动化审查结束的地方接手：架构合理性、产品契合度、代码可读性。

### CI 特定的 MCP 工具

在 GitHub Actions 环境中运行时，Claude Code 可以访问提供 CI 特定信息的额外 MCP 工具：

- `mcp__github_ci__get_ci_status` —— 获取某个提交的当前 CI 状态
- `get_workflow_run_details` —— 获取特定工作流运行的详细信息
- `download_job_log` —— 下载并读取特定 CI 任务的日志

这些工具使 Claude Code 能够在审查过程中直接读取 CI 失败日志——不仅仅是变更的代码，还包括失败的测试输出。一个既能查看代码又能查看测试失败的审查，比一个只能查看代码的审查更有用。

### 在 OpenTalon 中

在 OpenTalon 中，自动 PR 审查在每次修改 `cli/`、`api/` 或 `web/` 文件的拉取请求上运行。仅修改 `_bmad/artifacts/`（规划文档）或 `docs/` 的拉取请求跳过代码审查步骤——因为没有代码需要审查。ANTHROPIC_API_KEY 密钥设置在仓库的 CI 设置中。最大轮数限制为 5，防止在复杂 PR 上消耗过多 token——5 轮足够读取变更文件、检查 CLAUDE.md 和 architecture.md 的相关部分，并生成审查意见。
