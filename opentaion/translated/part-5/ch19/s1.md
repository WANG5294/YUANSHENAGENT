## 19.1 节：为 macOS 打包 CLI：Homebrew Tap

CLI 在开发者的机器上运行正常。要让它在陌生人的机器上也能正常运行，就需要打包——将开发代码库转换为可分发的制品，安装时无需配置 Python 环境。

目标体验：`brew install yourgithub/opentaion/opentaion` 安装 CLI，之后立即可以使用 `opentaion "help me debug this"`。不需要 `pip install`，不需要虚拟环境，不需要 PATH 配置。

### pyproject.toml 配置

CLI 的 `pyproject.toml` 需要分发配置：

```toml
# cli/pyproject.toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "opentaion"
version = "0.1.0"
requires-python = ">=3.12"
dependencies = [
    "click>=8.0",
    "rich>=13.0",
    "httpx>=0.27",
    "python-dotenv>=1.0",
    "tiktoken>=0.7",
]

[project.scripts]
opentaion = "opentaion.main:cli"

[tool.hatch.build.targets.wheel]
packages = ["src/opentaion"]
```

`[project.scripts]` 条目在安装包时创建 `opentaion` 命令。

### 构建分发包

```bash
cd opentaion/cli
uv build
```

输出：
```
dist/
├── opentaion-0.1.0-py3-none-any.whl
└── opentaion-0.1.0.tar.gz
```

`.whl` 文件是分发包。将其上传到 `opentaion/opentaion` 仓库的 GitHub Release。

### 创建 Homebrew Tap

Homebrew Tap 是一个名为 `homebrew-<name>` 的 GitHub 仓库。对于 OpenTalon：

1. 创建一个新的 GitHub 仓库：`yourgithub/homebrew-opentaion`
2. 创建公式文件：`homebrew-opentaion/Formula/opentaion.rb`

```ruby
# Formula/opentaion.rb
class Opentaion < Formula
  include Language::Python::Virtualenv

  desc "AI coding agent CLI with usage tracking"
  homepage "https://github.com/yourgithub/opentaion"
  url "https://github.com/yourgithub/opentaion/releases/download/v0.1.0/opentaion-0.1.0-py3-none-any.whl"
  sha256 "<sha256-of-the-wheel-file>"
  license "MIT"

  depends_on "python@3.12"

  resource "click" do
    url "https://files.pythonhosted.org/packages/.../click-8.1.7.tar.gz"
    sha256 "..."
  end

  # ... (uv generate-lockfile or pip-audit produces the resource list)

  def install
    virtualenv_install_with_resources
  end

  test do
    system bin/"opentaion", "--version"
  end
end
```

### 发布流程

当新版本准备就绪时：

1. 运行 `uv build` 并将 wheel 上传到 GitHub Releases
2. 计算 SHA256：`sha256sum dist/opentaion-0.1.0-py3-none-any.whl`
3. 使用新 URL 和 SHA256 更新 `opentaion.rb`
4. 推送到 `homebrew-opentaion`
5. Homebrew 用户运行 `brew upgrade opentaion`

发布流程可以自动化：一个在版本标签上触发的 GitHub Action，负责构建 wheel、创建 Release 并更新公式。

### 在 OpenTalon 中

在 OpenTalon 中，Homebrew tap 是 CLI 的主要分发渠道。它是"如何安装这个？"这个问题的答案——一条命令，无需管理依赖。对于更倾向于 pip 的开发者，PyPI 选项也存在，但对于非 Python 开发者会遇到的 macOS CLI 工具来说，Homebrew 是正确的首选选项。第 19.5 节的里程碑演练使用 Homebrew 安装作为全新用户旅程的入口点。
