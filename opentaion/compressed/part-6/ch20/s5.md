## 20.5 节：claude-code-router 模式

模型路由解决了一个模型选择本身无法解决的问题：根据每个请求的特征自动选择正确的模型，而无需开发者手动为每项任务选择。
`claude-code-router` 是一个拥有 25,000+ GitHub 星标的社区工具，它根据可配置规则将 Claude Code 请求路由到不同的模型提供商。对于 OpenTalon 用户，它实现了无需注意力的成本降低——正确的模型自动用于每项任务。
### claude-code-router 的作用

claude-code-router 位于 Claude Code 和模型提供商之间。它拦截每个请求，评估其路由规则，并将请求转发到适当的模型：
```
Claude Code 会话
    ↓
claude-code-router（评估路由规则）
    ↓
路由 1：简单任务 → DeepSeek R1（免费、快速）
路由 2：复杂推理 → Claude Sonnet 4.6（付费、能力强）
路由 3：样板代码 → Gemma 3 27B（免费、够用）
```

路由规则是可配置的：按任务类型（从提示内容检测），按估计复杂度（基于对话长度），或按预算（在达到成本阈值后停止用付费模型）。
### OpenTalon 的配置

```yaml
# .claude-code-router.yaml
routes:
  - name: "simple-queries"
    pattern: "explain|what is|list|show me"
    model: "deepseek/deepseek-r1:free"
    provider: "openrouter"

  - name: "implementation"
    pattern: "implement|write|create|add"
    model: "claude-sonnet-4-6"
    provider: "anthropic"
    max_tokens: 4096

  - name: "boilerplate"
    pattern: "generate|template|scaffold|dockerfile"
    model: "google/gemma-3-27b-it:free"
    provider: "openrouter"

  - name: "default"
    model: "deepseek/deepseek-r1:free"
    provider: "openrouter"
```

路由规则用简单的模式匹配来匹配用户的提示词。对于不想配置复杂规则的独立开发者，仅默认规则（除非其他规则匹配，否则用免费模型）就能为大多数不需付费模型能力的任务带来显著的成本降低。
### 向 CLI 添加 claude-code-router 支持

```python
# cli/src/opentaion/config.py (additions)
ROUTER_CONFIG_PATH = Path.home() / ".config" / "opentaion" / "router.yaml"

def load_router_config(config_path: Optional[Path] = None) -> Optional[dict]:
    """Load routing configuration if it exists."""
    path = config_path or ROUTER_CONFIG_PATH
    if path.exists():
        import yaml
        with open(path) as f:
            return yaml.safe_load(f)
    return None
```

```python
# cli/src/opentaion/main.py (additions)
@click.option("--router-config", type=click.Path(), default=None,
              help="Path to claude-code-router configuration file")
def cli(prompt, model, router_config):
    config = load_config()
    router = load_router_config(router_config)
    if router:
        model = select_model_from_router(router, prompt, model)
    # ... rest of CLI logic
```

想要自动路由的用户创建 `~/.config/opentaion/router.yaml`。不需路由的用户继续用 CLI，无需任何更改。
### 测量成本降低

优化前（基准会话——50 次 CLI 请求，混合任务）：
```
基准：
  50 次请求 × 2,000 Token 系统提示词 = 100,000 输入 Token 按全价
  无缓存、无路由
  所有请求使用 Sonnet 层模型
  估计成本：每次会话约 0.30 美元
```

优化后（相同会话，带缓存和路由）：
```
优化后：
  提示缓存：第一次请求全价，后续 49 次按 10%
    = 2,000 + (49 × 200) = 11,800 Token 用于系统提示词
  模型路由：60% 的请求路由到免费层
    = 30 次免费请求，20 次付费请求

  估计成本：每次会话约 0.09 美元
  降低幅度：约 70%
```

70% 的数字与里程碑目标一致。实际结果因会话组成和缓存命中率而异。
### 在 OpenTalon 中

在 OpenTalon 中，里程碑 M19 在第 20.4 节的提示缓存实现已部署，并且 `--router-config` 标志被添加到 CLI 时达成。前后对比测量在代表性的开发会话上进行。缓存和路由的综合降低效果确认了 70% 的目标。通过 Homebrew 安装 OpenTalon 的用户可以通过创建路由配置文件来启用路由——除了此处添加的标志支持外，无需对 CLI 代码更改。

### 刚刚发生了什么

两项有针对性的优化——缓存不变的内容（系统提示词）和将不需高级模型的任务（简单查询，样板代码）路由到免费替代品——将一个代表会话的成本降低了 70%。没有牺牲代码质量。路由规则确保实现任务仍用能力强的模型。缓存确保运行长时间会话的用户看到最大的成本降低，因为缓存的好处随着会话长度而累积。
