## 20.1 节：Token 经济学：为什么智能体系统很昂贵

单轮交互很便宜。智能体工作流则不然。同样一个在单次 API 调用中花费 0.002 美元的任务，以智能体方式执行时可能需 0.02-0.04 美元。这 10-20 倍的乘数并非异常——它是智能体系统工作方式的可预测结果。
理解其昂贵的原因是管理成本的前提。
### 三个成本驱动因素

**多轮上下文增长。** 多轮对话中的每一轮都会将之前的内容追加到后续轮次读取的上下文中。一个以 2,000 Token 提示词开始，每轮产生 500 Token 响应的会话具有：
- 第 1 轮：2,000 Token 输入
- 第 2 轮：2,500 Token 输入（第 1 轮对话现在在上下文中）
- 第 3 轮：3,000 Token 输入
- 第 10 轮：6,500 Token 输入

输入 Token 成本随着每一轮增长。对于有 20 轮的会话，每轮的平均输入成本大约是初始提示成本的 3 倍。智能体会话对于一个复杂任务通常运行 10-30 轮。
**工具调用冗长。** 工具输出通常很长。运行测试套件的 `Bash` 调用会返回每个测试结果——可能 200 行输出。`Grep` 调用会返回每个文件中每行匹配的结果。`Read` 调用会返回整个文件。每次工具调用的输出都会成为后续轮次上下文的一部分。
一个读取 10 个文件，运行 5 个测试命令并尝试 3 次编辑的实现会话，在任务完成之前会在上下文中积累 50,000-100,000 Token 的工具输出。
**自我纠正循环。** 当智能体产生不正确输出时，它会自我纠正。纠正需重新读取相关上下文，推理出错的原因并产生新输出。一个需 3 次自我纠正循环的任务，成本大约是无纠正情况下相同任务的 3 倍。
### 具体数字

一个典型的 OpenTalon 实现会话——一个 BMAD 故事，中等复杂度，3-4 小时的工作：
```
每次会话的近似 Token 消耗：
  系统提示词（CLAUDE.md）：    5,000 Token × 15 轮
  工具输出累积：               80,000 Token 总计
  模型响应：                   20,000 Token 总计
  自我纠正迭代：               2 次迭代 × 每次 15,000 Token

  估计会话总计：               ~200,000 Token
  按 API 定价：                ~0.20-0.60 美元（取决于模型层）
  按 Claude Code 订阅：        包含在月费中
```

这些是粗略估计，而非保证。包含多次自我纠正的复杂会话可能达到 500,000-2,000,000 Token。
### 订阅与 API 决策

Claude Code 提供两种定价模式：
- **订阅制**：月费固定，在速率限制内无限使用
- **API 模式**：按 Token 付费，无月度承诺

决策框架：
- 每天使用 Claude Code 超过 4-5 小时 → 订阅制可能更便宜
- 不定期使用或需要精确测量成本 → API 模式允许精确追踪
- 需要按项目归因成本的团队 → 使用按项目密钥的 API 模式

OpenTalon 的用量代理演示了 API 模式：成本被计量和可视化。跟随本书构建的独立开发者可以在获得几次会话的数据后，根据自己的实际用模式做出明智的决定。
### 在 OpenTalon 中

在 OpenTalon 中，Token 经济学在第 20.5 节的里程碑中变得具体：测量本章优化前后的成本降低。优化措施——模型选择，努力调优和提示缓存——专门针对这三个成本驱动因素。压缩减少了多轮上下文增长。子智能体委托（第 12 章）减少了父会话中的工具调用冗长度。努力校准通过为模糊任务分配更多思考时间，为说明充分的任务分配更少思考时间来减少自我纠正。
