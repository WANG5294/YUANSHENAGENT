## 第 12.4 节：大代码库策略

二十个文件时，谨慎的习惯就足够了。五十个文件时，它们成为强制性的。超过这个阈值，本章描述的策略不是可选的优化——它们决定了会话是完成工作，还是读取错误文件时耗尽上下文。

当第 11 章中三个并行构建被集成时，OpenTalon 达到了五十个文件。CLI、API 和 Web 共同产生了一个代码库，其中朴素的导航——顺序读取文件直到足够理解并采取行动——找到相关代码前就会失败。

### 域路由

第一个策略是最重要的：当一个组件中工作时，排除其他组件。

三个 OpenTalon 组件在操作上是独立的。一个实现新 CLI 命令的会话不须读取 FastAPI 路由处理器。一个调试 Web 仪表盘的会话不须读取 CLI 的上下文管理器。然而，若没有显式约束，Claude Code 会沿着依赖链任意跟踪——`cli/src/opentaion/main.py`中的导入可能引用共享类型文件中的类型，该文件又引用另一个文件，然后 Web 组件的目录就被读取了。

将以下指令添加到根 CLAUDE.md：

```markdown
## 域路由

当在 cli/ 中工作时，除非被明确要求解决集成问题，
否则不要读取 web/ 或 api/ 中的文件。

当在 api/ 中工作时，除非被明确要求解决集成问题，
否则不要读取 cli/ 或 web/ 中的文件。

当在 web/ 中工作时，除非被明确要求解决集成问题，
否则不要读取 cli/ 或 api/ 中的文件。

跨组件问题应通过读取集成规格说明来解决，
而不是通过同时读取两个组件。
```

这条指令并不阻止跨组件工作。它要求这些工作必须是显式的。一个实现新 CLI 命令且碰巧触及 API 合约的会话仍然可以跨越边界——但必须被要求这么做，而非允许随意漂移到那里。

### 先 Grep 再读取

每次文件读取的成本与文件长度成正比。为发现相关函数不在其中而读取一个文件，却没有任何回报地花费了这些 token。

Grep 比读取便宜两到三个数量级。一个标识函数定义文件和行号的 grep 结果花费大约 100 token。读取错误的 500 行文件花费 5,000 token。

大代码库的正确导航模式：

```
1. 用 Grep 搜索符号、函数或概念
2. 从 grep 结果中识别文件和行号
3. 只读取该文件，从相关行开始
4. 仅在必要时跟踪引用
```

错误的模式：

```
1. 读取入口点文件
2. 读取它导入的文件
3. 读取那些文件导入的文件
4. 最终找到相关代码
```

第二种模式小型代码库中有效，因为没有多少文件可读。五十个文件时，它找到相关代码前就耗尽了上下文。

CLAUDE.md 可以显式强制实施这一点：

```markdown
## 导航规则

始终先 Grep 再读取。在读取任何文件之前，使用 Grep 确认
它包含相关代码。不要投机性地读取文件。
```

### 接口优先的导航

合约比实现小。函数签名是 3 行。实现其函数体可能是 50 行。类的公共 API 是 10 行。实现其私有方法可能是 200 行。

导航大代码库以理解系统时，先读取接口再读取实现：

- 先读取`__init__.py`文件——它们暴露模块的公共 API
- 读取使用类型的文件前先读取类型定义文件
- 先读取测试文件再读取实现文件——测试用每个测试用例约 100 token 描述代码应该做什么；实现可能每个行为有 20 行

回报：理解模块做什么通常只需要其接口。理解它如何做到则须读取实现。许多任务只需要前者。

### 分块读取长文件

有些文件抗拒这种纪律。一个 1,500 行的 FastAPI 路由文件可能确实需要完整读取——所有的路由处理器都一个文件中，而任务需要理解它们如何交互。

这些情况，使用显式偏移量分块读取：

```
读取 api/routers/v1.py，第 1–100 行（导入和路由注册）
读取 api/routers/v1.py，第 100–300 行（认证端点）
读取 api/routers/v1.py，第 300–600 行（使用量代理端点）
```

分块读取而非一次性读取，允许会话在找到相关部分后立即停止。若任务涉及使用量代理，可能根本不须读取第 1–300 行。

### 在 OpenTalon 中

OpenTalon 中，这些策略在三个组件合并的那一刻开始应用。域路由指令在集成会话期间被加入根 CLAUDE.md。先 Grep 再读取的规则被添加到同一文件中。当代码库第五部分中后来增长到 100 个文件以上时——增加了部署配置、故事文件、测试夹具——同样的规则无需修改即可扩展。50 个文件时建立的良好习惯代码库翻倍时不会被替换；它们成为使进一步增长可控的承重结构。第 12.5 节通过黄金集验证它们。
