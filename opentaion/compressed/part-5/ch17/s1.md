## 17.1 节：无头 Claude Code：标志与 JSON 输出

Claude Code 是为交互式用而设计的——开发者输入，智能体响应，如此往复。但让它在终端会话中有用的那些能力，同样让它在 CI 流水线中有用。当 Claude Code 以无头模式运行时，它可以审查代码，分析失败，生成文档并产生机器可解析的输出，无需任何人参与循环。
从交互式到无头用的转变只需一小套标志。
### 核心标志

**`claude -p "提示词"`** —— 非交互式单次执行。Claude Code 处理提示词，执行所有工具调用，完成后退出。没有对话历史，没有 REPL，不等待用户输入。这是 CI 流水线的主要模式。
```bash
# 在 CI 中审查更改过的文件
claude -p "Review the changes in this PR. Check against the \
OpenTalon coding conventions in CLAUDE.md. List any issues \
as a numbered list. If no issues found, output 'OK'."
```

**`--output-format=json`** —— 为机器解析提供结构化输出。Claude Code 不是输出 Markdown 文本，而是输出一个包含智能体响应和元数据的 JSON 对象。
```bash
claude -p "Analyze the test failures below and return a JSON object \
with: {cause, affected_files, recommended_fix}" \
--output-format=json \
< test-output.txt
```

**`--max-turns N`** —— 限制智能体的迭代次数。CI 审查任务应在 3-5 轮内完成；如果运行了 20 轮，肯定有问题。设置 `--max-turns 5` 可防止任务失控。
**`--no-interactive`** —— 明确阻止 Claude Code 提示输入。在 CI 环境中，任何输入提示都会导致任务无限挂起。此标志使失败显式化而非静默化。
**`--dangerously-skip-permissions`** —— 在隔离环境中绕过所有权限提示。这是唯一适合用此标志的场景：CI 运行器的整个执行环境是临时的，代码库是全新克隆的。绝不要在开发者的机器上用它。
**`--system-prompt "..."`** —— 覆盖默认的系统提示词。适用于需特定角色扮演的 CI 任务——如只关注安全性的代码审查员，或仅以特定格式输出的变更日志生成器。
### 管道输入

Claude Code 接受标准输入，以支持可组合的管道：
```bash
# 将测试失败信息路由到 Claude Code 进行分析
uv run pytest --tb=short 2>&1 | \
  claude -p "Explain the root cause of each failing test. \
             Be concise. Format as: Test Name: root cause." \
  --no-interactive

# 将文件路由进行审查
cat api/routers/proxy.py | \
  claude -p "Review this file for security issues. \
             Focus on input validation and error handling."
```

管道输入的内容成为提示语境的一部分。Claude Code 在处理 `-p` 提示词之前先读取标准输入。
### JSON 输出结构

当指定 `--output-format=json` 时：
```json
{
  "type": "result",
  "subtype": "success",
  "result": "1. Missing input validation...\n2. Error message exposes...",
  "session_id": "abc-123",
  "total_cost_usd": 0.0023,
  "turns": 3
}
```

CI 流水线可以解析此输出：提取结果用于 PR 评论，根据预算限制检查 `total_cost_usd`，或在继续之前确认 `subtype` 为成功。
### 在 OpenTalon 中

在 OpenTalon 中，无头 Claude Code 出现在三个 CI 场景中：自动 PR 审查（第 17.2 节），变更日志生成（第 17.4 节）和部署门控（第 17.5 节）。每个场景都用带有特定提示词的 `claude -p` 和 `--output-format=json` 来产生机器可解析的结果。`-p` 标志和 `--no-interactive` 是不可妥协的组合：没有 `--no-interactive`，遇到意外输入提示的流水线任务会一直挂起，直到运行器超时。
