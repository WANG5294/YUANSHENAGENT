## 19.4 节：使用魔法链接的电子邮件注册

认证系统在第 14 章设计，在第 16 章实现。本节提供了生产部署的完整图景：Supabase 配置，React 实现以及使整个流程工作的会话管理。
### 为什么使用魔法链接

用魔法链接而非密码的决定，并非为了便利——它是一个消除了整类风险的工程权衡。
密码认证需：正确存储哈希密码（用适当的成本因子的 bcrypt），实现密码重置流程，处理凭证填充攻击，教育用户关于密码强度的知识。每一个都是安全出错的隐患。
魔法链接需：发送电子邮件（Supabase 处理），验证时间限制的 Token（Supabase 处理），创建会话（Supabase 处理）。开发者的工作范围是：一个电子邮件输入字段，一个提交按钮和一个回调处理器。这是 20 行代码，而不是 200 行。
权衡在于：用户需访问其电子邮件才能登录。对于独立开发者工具，这是可以接受的。用工作邮箱注册的用户始终可以登录。没有需记住的密码，没有因错误密码尝试而被锁定的账户。
### Supabase 配置

在 OpenTalon 项目的 Supabase 仪表盘中：
1. Authentication → Email → 启用魔法链接登录
2. Authentication → Email → 自定义魔法链接邮件模板：

```
Subject: Sign in to OpenTalon

Body:
Hi,

Click the link below to sign in to your OpenTalon account.
This link expires in 1 hour.

{{ .ConfirmationURL }}

If you didn't request this email, you can safely ignore it.
```

3. Authentication → URL Configuration → Site URL：`https://opentaion.vercel.app`
4. Authentication → URL Configuration → Redirect URLs：添加 `https://opentaion.vercel.app/auth/callback`

### React 实现

```typescript
// web/src/lib/supabase.ts
import { createClient } from '@supabase/supabase-js'

export const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY
)
```

```typescript
// web/src/App.tsx — the complete auth-aware app structure
import { useState, useEffect } from 'react'
import { supabase } from './lib/supabase'
import type { Session } from '@supabase/supabase-js'
import { RegistrationPage } from './pages/RegistrationPage'
import { Dashboard } from './pages/Dashboard'

export default function App() {
  const [session, setSession] = useState<Session | null>(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    // Get initial session
    supabase.auth.getSession().then(({ data: { session } }) => {
      setSession(session)
      setLoading(false)
    })

    // Listen for auth state changes (handles magic link callback)
    const { data: { subscription } } = supabase.auth.onAuthStateChange(
      (_event, session) => {
        setSession(session)
      }
    )

    return () => subscription.unsubscribe()
  }, [])

  if (loading) return <div>Loading...</div>

  return session ? <Dashboard session={session} /> : <RegistrationPage />
}
```

此即完整的认证条件渲染。未经身份验证的用户看到 RegistrationPage。已认证用户看到 Dashboard。当用户点击魔法链接时，Supabase 的 JS 客户端处理 Token 交换，并通过新会话调用 `onAuthStateChange`——应用程序自动转换到仪表盘，无需任何额外的路由逻辑。
### 会话管理

Supabase 自动处理会话刷新。访问 Token 每小时过期一次；Supabase 的客户端库用刷新 Token 静默刷新它。开发者无需实现会话过期逻辑。
登出只需三行代码：
```typescript
await supabase.auth.signOut()
// onAuthStateChange fires with session = null
// App re-renders to RegistrationPage
```

### 在 OpenTalon 中

在 OpenTalon 中，魔法链接流程在用户打开邮件的所有设备上都能工作。如果用户在笔记本电脑上注册，在手机上打开魔法链接，手机浏览器会获得会话。Supabase 支持每个用户多个并发会话——笔记本电脑和手机是独立的会话，都有效，都能访问仪表盘。对于独立开发者工具，这是无需任何额外实现的正确行为。
