## 19.3 节：OpenRouter 集成：多模型支持

OpenRouter 是 OpenTalon 代理背后的 LLM 提供商。它将数十种模型（包括几个具有免费层的模型）聚合在一个与 OpenAI 兼容的 API 之后。跟随本书构建的读者，这意味着无需前期基础设施成本，无需信用卡即可达到第一个可工作状态。
### 免费模型层

OpenRouter 维护一个免费模型列表，这些模型不消耗积分。列表会随着提供商添加和移除免费服务而变化；当前列表始终可以在 `openrouter.ai/models?free=true` 获取。在撰写本文时，稳定可用的免费模型包括：
- `meta-llama/llama-3.3-70b-instruct:free` —— 强推理能力，128K 上下文
- `deepseek/deepseek-r1:free` —— 扩展思考能力，适合代码
- `mistralai/mistral-7b-instruct:free` —— 更快、更轻量，适合简单任务
- `google/gemma-3-27b-it:free` —— 平衡的能力

`:free` 后缀是 OpenRouter 对模型免费版本用的约定。有些模型有付费和免费两种变体，具有不同的速率限制。
### 模型字符串格式

OpenRouter 模型字符串用 `provider/model-name` 格式。四个推荐模型是：
```python
# cli/src/opentaion/config.py
FREE_MODELS = [
    "meta-llama/llama-3.3-70b-instruct:free",
    "deepseek/deepseek-r1:free",
    "mistralai/mistral-7b-instruct:free",
    "google/gemma-3-27b-it:free",
]

DEFAULT_MODEL = FREE_MODELS[0]  # Llama 3.3 70B as the default
```

CLI 接受 `--model` 来覆盖默认值：
```bash
opentaion --model deepseek/deepseek-r1:free "explain this bug"
```

### 回退逻辑

当主模型不可用时（速率受限，临时宕机），API 会尝试回退列表中的下一个模型：
```python
# api/src/opentaion_api/routers/proxy.py (partial)
async def proxy_with_fallback(
    request_body: dict,
    api_key_record: ApiKey,
    models: list[str]
) -> dict:
    for model in models:
        try:
            response = await openrouter_client.complete(
                {**request_body, "model": model}
            )
            return response
        except OpenRouterRateLimitError:
            if model == models[-1]:
                raise  # Last model also rate limited
            continue  # Try next model
```

回退对 CLI 是透明的。CLI 请求了 `deepseek/deepseek-r1:free`；如果该模型速率受限，API 会尝试 `mistralai/mistral-7b-instruct:free` 而 CLI 毫不知情。无论哪个模型实际处理了请求，响应都以相同的格式返回。
### OpenAI 兼容性

OpenRouter 在 `https://openrouter.ai/api/v1/chat/completions` 接受 OpenAI 格式的请求。请求体：
```json
{
  "model": "meta-llama/llama-3.3-70b-instruct:free",
  "messages": [
    {"role": "system", "content": "You are a helpful coding assistant."},
    {"role": "user", "content": "Explain this function."}
  ],
  "stream": true
}
```

响应是 OpenAI 格式的：非流式时为 `choices[0].message.content`，流式时为服务器发送事件（SSE）。OpenTalon CLI 的 `OpenRouterClient` 类用这种格式。当 CLI 配置为调用本地代理而非直接调用 OpenRouter 时，它会向 `POST http://localhost:8000/v1/chat/completions` 发送相同的请求格式，代理再将其转发给 OpenRouter。
### 在 OpenTalon 中

在 OpenTalon 中，OpenRouter 集成是这个系统可以免费开始用的原因。按照本书操作的读者可以零 API 支出时获得一个可工作的 CLI 智能体：创建一个 OpenRouter 账户（无需信用卡），获取一个 API 密钥，在环境变量中设置它，然后默认模型（Llama 3.3 70B）会在零成本下处理所有请求。这是从第 4 章的 CLAUDE.md 开始塑造了整个技术栈的不可妥协的设计约束。
