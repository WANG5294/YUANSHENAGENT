## 19.5 节：Token 用量追踪：模式、计量和仪表盘

用量追踪系统是 Web 平台存在的原因。没有它，OpenTalon 只是一个调用 OpenRouter 代理的 CLI 智能体——有用，但没有特色。有了它，用户拥有了其他免费 CLI 智能体所没有的东西：按天和按模型对其 Token 消耗的实时可见性。
本节追踪从 CLI 调用到仪表盘显示的完整数据流。
### 完整的计量流程

```
1. 用户运行：opentaion "find and fix the type error in auth.py"

2. CLI 构建 OpenAI 格式的请求：
   POST http://opentaion-api.up.railway.app/v1/chat/completions
   X-Api-Key: opentaion_abc12345...
   Body: { "model": "meta-llama/llama-3.3-70b:free",
           "messages": [...] }

3. API 验证 API 密钥（针对 api_keys 表进行 bcrypt 检查）

4. API 向 OpenRouter 发起代理请求：
   POST https://openrouter.ai/api/v1/chat/completions
   Authorization: Bearer ***
   Body: (same request body)

5. OpenRouter 流式返回响应

6. API 实时将响应体流式传输给 CLI

7. 流式传输完成后，最终的 SSE 事件包含：
   data: {"usage": {"prompt_tokens": 1240, "completion_tokens": 318}}

8. API 写入 usage_logs（作为后台任务）：
   INSERT INTO usage_logs
   (user_id, model, input_tokens, output_tokens, created_at)
   VALUES ($1, $2, $3, $4, now())

9. CLI 显示流式传输的响应
```

第 6 步和第 8 步是并发的——CLI 看到响应的同时，API 正在写入用量日志。后台任务确保日志写入不会给流式响应增加延迟。
### 带有索引的 UsageLog 模式

```sql
CREATE TABLE usage_logs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID NOT NULL REFERENCES auth.users(id),
    model           VARCHAR(100) NOT NULL,
    input_tokens    INTEGER NOT NULL CHECK (input_tokens >= 0),
    output_tokens   INTEGER NOT NULL CHECK (output_tokens >= 0),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Index for dashboard queries (user's usage by time)
CREATE INDEX idx_usage_logs_user_time
ON usage_logs (user_id, created_at DESC);

-- Index for model breakdown queries
CREATE INDEX idx_usage_logs_user_model
ON usage_logs (user_id, model, created_at DESC);

-- Row-level security
ALTER TABLE usage_logs ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users read their own logs"
ON usage_logs FOR SELECT
USING (auth.uid() = user_id);
```

这些索引使两种最常见的仪表盘查询在大规模下也能保持快速。
### UsageChart 组件

```tsx
// web/src/components/UsageChart.tsx
import { BarChart, Bar, XAxis, YAxis, Tooltip, ResponsiveContainer } from 'recharts'

interface DailyUsage {
  date: string
  total_tokens: number
}

interface Props {
  data: DailyUsage[]
}

export function UsageChart({ data }: Props) {
  return (
    <ResponsiveContainer width="100%" height={240}>
      <BarChart data={data} margin={{ top: 0, right: 0, bottom: 0, left: 0 }}>
        <XAxis
          dataKey="date"
          tickFormatter={(d) => new Date(d).toLocaleDateString('en-US',
            { month: 'short', day: 'numeric' }
          )}
          tick={{ fontSize: 12 }}
        />
        <YAxis tick={{ fontSize: 12 }} width={60} />
        <Tooltip
          formatter={(value: number) => [
            value.toLocaleString() + ' tokens', 'Total'
          ]}
          labelFormatter={(label) => new Date(label).toLocaleDateString(
            'en-US', { weekday: 'long', month: 'long', day: 'numeric' }
          )}
        />
        <Bar dataKey="total_tokens" fill="#2563eb" radius={[2, 2, 0, 0]} />
      </BarChart>
    </ResponsiveContainer>
  )
}
```

此即完整的组件——35 行代码。它根据 `/usage/daily` API 响应渲染一个柱状图，带有显示日期和 Token 计数的悬停提示。
### 新用户旅程

在系统部署后，完整的新用户旅程：
1. 访问 `https://opentaion.vercel.app`
2. 输入邮箱 → 收到魔法链接 → 点击链接 → 到达空仪表盘
3. 导航到 API Keys → 创建密钥 → 复制显示的密钥
4. 安装 CLI：
```bash
  brew tap yourgithub/opentaion
  brew install opentaion
  ```
5. 配置 CLI：
```bash
  export OPENTAION_API_URL=https://opentaion-api.up.railway.app
  export OPENTAION_API_KEY=opentaion_abc12345...
  ```
6. 运行第一个命令：
```bash
  opentaion "what files are in the current directory?"
  ```
7. 刷新仪表盘——用量出现

从注册到首次用：大约 5 分钟。这是开发者工具的正确基准。如果超过这个时间，说明加载流程中有些地方需在推广产品之前修复。
