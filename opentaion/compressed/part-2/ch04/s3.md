## 章节 4.3：单仓库策略：一个项目三个 CLAUDE.md 文件

层次化加载系统存在为解一特定问题：项目非统一。Python 规则不适用于 TypeScript。FastAPI 路由约定与 Click 命令定义无关。单一 CLAUDE.md 试图覆盖所有时，要么覆盖差（对各领域太单薄），要么产生不可读文档，在首次工具调用前耗完上下文预算。

OpenTalon 有三组件：Python CLI、React Web 仪表盘、FastAPI 后端。它们共享项目标识、部署故事和彼此关系——但实现约定完全独立。正确结构是三 CLAUDE.md 文件，处于有意的层级中。

### 根 CLAUDE.md：什么是通用的

根 `CLAUDE.md`（第 2 章编写）处理对项目整体成立的一切：

- **项目标识**：OpenTalon 是什么、解决什么问题、书与软件如何交织
- **共享技术栈**：已批准库完整列表及不能偏离的规则
- **会话前检查清单**：Claude Code 写任何东西前运行的五步骤协议
- **通用规则**："不要发明尚未构建的代码"；"不要跳过写作前检查清单"
- **里程碑地图**：哪章产出什么可交付成果，使上下文始终接地

根 CLAUDE.md 不指定 Python 格式化偏好、TypeScript 严格性设置或 FastAPI 端点命名约定。那些放组件文件中——因只在处理该组件时才相关。

### cli/CLAUDE.md：Python 领域

Claude Code 打开 `cli/` 时，加载根 CLAUDE.md 并加 CLI 规则。`cli/CLAUDE.md` 覆盖：

```markdown
# CLI — 组件规则

## 环境
- Python 3.12+。包管理器：uv。绝不要直接使用 pip。
- 运行命令：智能体循环使用 `uv run python -m opentaion`
- 运行测试：`uv run pytest tests/` —— 始终从 cli/ 目录执行

## Click 约定
- 命令使用 @click.command() 和 @click.option() 装饰器
- 命令分组在 `cli = click.group()` 下
- 通过 Click 上下文（ctx.obj）传递配置，而非全局状态

## Rich 显示
- 控制台输出通过 `rich.console.Console()` 进行
- 状态微调器：`with console.status("[bold]Working..."):`
- 错误输出：`console.print("[red]Error:[/red] ...", err=True)`

## 反模式
- 绝不从 web/ 或 api/ 导入——CLI 是独立工具
- 绝不使用同步 httpx——所有 HTTP 调用使用 async/await
- 绝不硬编码 API 端点——从配置读取
```

此文件简短、具体、可操作。条规则有直接行为后果。Claude Code 写 Click 命令时拥有所需 Python 约定，不看到不需的 Vite 构建配置。

### web/CLAUDE.md：TypeScript 领域

`web/CLAUDE.md` 承载前端规则：

```markdown
# Web — 组件规则

## 环境
- Vite 5.x + React 18 + TypeScript（严格模式）
- 运行：`npm run dev` —— 开发服务器
- 构建：`npm run build` —— 生产构建
- 需要 Node.js 20+

## 组件约定
- 函数组件配合 TypeScript 类型——无类组件
- 状态：useState 用于局部状态，无 Redux 或 Zustand
- 样式：仅使用 Tailwind 工具类——无自定义 CSS 文件
- 图表：所有数据可视化使用 Recharts 库

## 文件结构
- src/components/ —— 可复用组件
- src/pages/ —— 顶层路由组件（Login、Dashboard）
- 类型在使用它们的文件中定义，不在 types/ 目录中

## 反模式
- 绝不使用内联样式——仅使用 Tailwind
- 绝不在 web 代码中访问 api/ 或 cli/ 目录
- 绝不添加 npm 包而不先检查 Tailwind 或 Recharts 是否已满足需求
```

同样简短、具体、可操作。无 TypeScript 教程，无解释 Tailwind。

### api/CLAUDE.md：FastAPI 领域

```markdown
# API — 组件规则

## 环境
- Python 3.12+ / FastAPI 0.110+
- 运行：`uv run uvicorn main:app --reload`
- 数据库：Supabase PostgreSQL（异步 SQLAlchemy）

## 路由约定
- 所有路由返回 Pydantic 模型，而非字典
- 端点名称：名词优先、连字符连接（`/api/usage-records`）
- 认证：通过 `Depends(get_current_user)` 提取 Token——不手动读取头部

## SQLAlchemy 模式
- 所有数据库操作使用异步会话：`async with AsyncSession() as session:`
- 模型放在 models/ 目录中，每个资源一个文件

## 反模式
- 绝不提交数据库连接字符串——使用环境变量
- 绝不编写原生 SQL——使用 SQLAlchemy 查询 API
- 绝不返回包含错误体的 200 状态码——正确使用 HTTP 状态码
```

### 实践中的继承模型

Claude Code 修复 `api/routes/auth.py` 中 Bug 时，有效上下文包括：

1. 根 CLAUDE.md —— 项目标识、共享技术栈、通用规则
2. `api/CLAUDE.md` —— FastAPI 约定、SQLAlchemy 模式、反模式

不加载 `cli/CLAUDE.md` 或 `web/CLAUDE.md`。它们不相关，会是噪音。加载层级将上下文过滤为与当前位置相关的内容。

非次要便利。不相关上下文比缺失更糟，因可能产生不相关建议。写 TypeScript 时加载 Python 约定的 CLAUDE.md，可能产生 Python 风格 TypeScript。子目录分离防止交叉污染。

实用规则：当子目录有根文件无法在不臃肿下覆盖的真实、领域特定约束时，创建组件 CLAUDE.md。勿为每子目录都创建——`api/` 内部 `models/CLAUDE.md` 几乎肯定不必。OpenTalon 的三组件是恰当粒度。

### OpenTalon 连接

OpenTalon 中，三组件 CLAUDE.md 尚不存在——代码库本身也还不存在。它们将在第 4.5 节里程碑 M3 中被创建，届时建完整生产级 CLAUDE.md 结构。写这些文件前，下两节涵盖包含什么和排除什么，及使 CLAUDE.md 失败的反模式。这些原则在实际编写时指导内容——顺序重要。
