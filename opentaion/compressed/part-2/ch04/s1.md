## 章节 4.1：层次化加载：Claude Code 如何读取记忆

第一部分确认 Claude Code 无会话记忆——每会话从空白开始。它拥有的是文件。第 2 章编写的 CLAUDE.md 是系统提供的最接近持久化记忆之物。第二部分精确考察此记忆如何加载、可采何结构、以及制约它的约束。本章从机制开始：Claude Code 如何读 CLAUDE.md、加载哪些文件、按何顺序。

"层次化（hierarchical）"精确。Claude Code 非简单找并加载仓库中每个 CLAUDE.md，而是按定义的优先级顺序加载，子目录文件是附加式的（additive）——扩展而非替换上层。理解此顺序是设计 OpenTalon 三文件 CLAUDE.md 结构的先决条件。

### 五个层级，一个顺序

Claude Code 在五加载级别处理 CLAUDE.md，从最高到最低优先级：

**1. 托管策略（组织级别）。** 若 Claude Code 在 Anthropic 组织账户内部署，管理员可设适用于所有用户的策略文件，无法被项目设置排除。个人订阅的独立开发者永不遇到此级别。它存在以支持需安全团队保证约束的企业部署。

**2. 用户级别（`~/.claude/CLAUDE.md`）。** 用户主目录中单文件。放置适用于所有项目的个人偏好：偏好的编程语言习惯、提醒不在代码注释中用表情符、本地环境特殊性说明。每项目每会话加载。

**3. 项目根目录（`./CLAUDE.md`）。** 第 2 章编写的文件。主项目宪法——描述项目是什么、技术栈、制约规则。Claude Code 打开项目目录时加载。

**4. 子目录 CLAUDE.md 文件。** Claude Code 在 `cli/` 中工作时加载 `cli/CLAUDE.md` 作为根 CLAUDE.md 补充。移到 `api/` 则加载 `api/CLAUDE.md`。这些是领域特定文件：携带仅处理该组件时才适用的规则。不替换根文件，两者同时活动。

**5. `.claude/rules/` 拆分文件。** 对根 CLAUDE.md 可能变得过长的项目，支持将文件拆为 `.claude/rules/` 中多文件。该目录中文件与根 CLAUDE.md 一起加载。组织模式非优先级级别——所有 `.claude/rules/` 文件视为项目根级别部分。

文件冲突时优先级重要。用户级规则可被项目根覆盖。项目根规则可被子目录覆盖。最具体文件获胜。

### "附加式"在实践中意味什么

"附加式"易被误解为"累积到膨胀"。更好的思维模型是分层：每层添加领域特定规则而不使下层失效。

Claude Code 打开 OpenTalon `cli/` 子目录时，其有效指令集是三文件的并集：用户级 `~/.claude/CLAUDE.md`（若有）、项目根 `CLAUDE.md` 和 `cli/CLAUDE.md`。全部三活动。cli/ 文件添加 Python 特定规则——"用 uv 别用 pip；偏好 Click 装饰器定义命令；用 `uv run pytest` 运行测试"——无需重复根文件内容。根文件仍说"技术栈固定"。cli/ 文件说"此技术栈的 CLI 组件所需"。各层互相补充。

对 OpenTalon 这样的单仓库重要，其中三组件（CLI、Web、API）共享项目标识但有完全不同工具约定。试图覆盖所有三领域的单一根 CLAUDE.md 会冗长不可读——且更糟，即使只处理 CLI 也加载所有三套规则，使上下文充满不相关指令。子目录模式解决：只加载与你当前工作位置相关的内容。

### @-引用语法

与层次化加载分开，Claude Code 支持用 `@` 语法的显式文件引用。任何 CLAUDE.md 中：

```
@docs/architecture.md
```

会使 Claude Code 处理该 CLAUDE.md 时将该文件内容加载到上下文。内联引用——被引用文件内容出现在 `@` 位置，像写在那一样。

与子目录加载区别在加载时间点。子目录 CLAUDE.md 在你导航到该目录时自动加载。@-引用文件在处理含引用的 CLAUDE.md 时加载。两者都在会话开始时发生，但触发条件不同。第 4.2 节全面涵盖 @-引用系统，包括条件加载模式。

### claudeMdExcludes 设置

额外控制：项目 Claude Code 设置中 `claudeMdExcludes` 字段接受 Glob 模式。匹配文件在加载中被跳过。当某子目录 CLAUDE.md 不应在特定会话加载时，或外部包含你希望抑制的 CLAUDE.md 时有用。设置位于项目级 `.claude/settings.json`。

### OpenTalon 连接

在 OpenTalon 中，层次化加载系统是我们将在本章逐步构建的架构。第 2 章的根 `CLAUDE.md` 处理项目标识、共享技术栈约束和会话前检查清单。第 4 章将添加 `cli/CLAUDE.md`、`web/CLAUDE.md` 和 `api/CLAUDE.md`——各含其子目录所需组件特定规则。Claude Code 构建 CLI 智能体时拥有根规则加 Python 规则；构建 Web 仪表盘时拥有根规则加 Vite/React/Tailwind 规则。无组件 CLAUDE.md 需重复根文件已说内容。加载层级自动组合。

每个文件应含什么——以及应排除什么——是接下来几节要回答的问题。在此之前，第 4.2 节机制解释如何从 CLAUDE.md 内部引用文件，而非仅依赖基于目录的加载。
