## 第 6.4 节：通知钩子：操作系统级提醒

前两节增了代理循环内自动化：对 Claude Code 能做什么的约束，及对其已完工作的质量检查。通知钩子性质不同。它不影响代码或工具调用，而影响开发者工作流——消除盯着终端看的需要。

长代理任务需时间。从代码库扫描建规范文档、据测试套件实现完整 CLI 命令、调试多文件问题——这些运行五到十五分钟。开发者盯着终端不做任何有用事。但若切其他应用，错过 Claude Code 完成并请求输入的时刻，会话闲置等待。

Notification 钩子即解决方案。Claude Code 想提醒用户时触发——代理等输入、长任务完成、错误需关注。钩子触发 OS 级 macOS 通知。开发者屏上出现通知，切换回终端，会话继续。

### macOS 实现

macOS 自带 `osascript`，AppleScript 执行器。无第三方工具，无额外安装。一行命令触发原生通知：

```bash
osascript -e 'display notification "Claude Code needs your attention" with title "Claude Code" sound name "Glass"'
```

完整通知钩子：

```bash
#!/bin/bash
# .claude/hooks/notify-macos.sh

INPUT=$(cat)
MESSAGE=$(echo "$INPUT" | python3 -c "
import json, sys
d = json.load(sys.stdin)
print(d.get('message', 'Claude Code needs your attention'))
" 2>/dev/null || echo "Claude Code needs your attention")

osascript -e "display notification \"$MESSAGE\" with title \"Claude Code\" sound name \"Glass\""
```

settings.json：

```json
{
  "hooks": {
    "Notification": [
      {
        "matcher": ".*",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/notify-macos.sh"
          }
        ]
      }
    ]
  }
}
```

`.*` 匹配器触发所有 Notification 事件。通知含 Claude Code 传递的消息——通常描述它需要什么或完成了什么。

### 工作流程的变化

实际影响显著。无通知钩子时，10 分钟任务要么需盯着终端，要么需定期回查。有钩子，工作流变：启动任务，最小化 Claude Code 窗口到后台，做其他工作，收通知，切回来。终端从焦点变背景。

对 OpenTalon 构建——含几个长合成任务如写 SPEC.md、运行完整测试套件、构建完整 Web 仪表板——此钩子将十五分钟等待从阻塞间隔变后台操作。

`sound name "Glass"` 播放 Glass 声。可改成任何系统声音名（Basso、Blow、Bottle、Frog、Funk、Hero、Morse、Ping、Pop、Purr、Sosumi、Submarine、Tink），或移 `sound name` 子句获静默通知。

### 扩展模式

同样通知钩子可扩展至其他渠道。团队环境开发者可路由到 Slack webhook，长任务完成时向频道发消息。钩子结构支持 HTTP 类型——将 `command` 替换为 `http` 并提供端点。相同事件，不同传递机制。

对 OpenTalon 单人开发，macOS 通知足够。无团队频道，反馈周期紧，OS 通知覆盖所有场景。CI/CD 章节（第 17 章）将模式扩展到 GitHub Actions 通知。

### 与 OpenTalon 的关联

OpenTalon 中，通知钩子在 BMAD 辅助阶段（第四部分）尤有价值——单任务"分析 PRD 并写架构文档"可能运行 15-20 分钟。无通知时需持续终端关注。有钩子，架构合成运行时开发者可做其他事，通知在文档就绪时发信号。钩子从第 6 章就位，故每长会话都受益。
