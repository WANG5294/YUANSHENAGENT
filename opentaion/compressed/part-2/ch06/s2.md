## 第 6.2 节：PreToolUse 钩子：强制执行约束

PreToolUse 钩子是一道闸门，立于工具调用请求与其执行间。Claude Code 决定运行 Bash 命令时，钩子在命令运行前触发。钩子检查命令，返回决策。若"阻止"，命令永不执行。此结构性强制执行——非 Claude Code 可能忘记的指南，而是它无法通过的闸门。

此区别至关重要。CLAUDE.md 中规则是 Claude Code 读并以不同精确度遵循的指令。PreToolUse 钩子是种检查，无论 CLAUDE.md 写什么、会话对话上下文是什么、自上次读 CLAUDE.md 以来累积多少工具调用，它都会运行。闸门不疲倦。

### 三种钩子响应

PreToolUse 钩子通过 stdin 以 JSON 收工具名称及输入，返回 JSON 响应，三形态：

**批准（Approve）** —— 工具调用不变继续：
```json
{"decision": "approve"}
```

**阻止（Block）** —— 工具调用取消，返回错误：
```json
{"decision": "block", "reason": "Bash command contains rm -rf outside allowed directories"}
```

Claude Code 收原因，纳入推理，决定如何继续——通常尝试其他方式或询问用户。

**修改（Modify）** —— 工具调用继续，但输入已改：
```json
{"updatedInput": {"command": "ls -la /safe/directory"}}
```

Modify 最灵活也最易因粗心带来危险。静默重写 Bash 命令的钩子产生意外行为。谨慎用 modify，仅转换明确无歧义时。

### 匹配器语法

钩子配置中 `matcher` 决定哪些工具调用触发钩子。常见模式：

- `"Bash"` —— 匹配所有 Bash
- `"Write|Edit"` —— 匹配 Write 和 Edit（竖线是正则"或"）
- `"Bash(rm -rf*)"` —— 匹配命令含 `rm -rf*` 的 Bash
- `".*"` —— 匹配所有（少用，开销大）

匹配器针对工具名评估，对 Bash 则对命令字符串。用覆盖约束的最具体匹配器。

### 示例：阻止危险的 Shell 命令

以下钩子阻止含 `rm -rf` 的 Bash 命令，当影响已定义安全区域外目录时：

```bash
#!/bin/bash
# .claude/hooks/block-dangerous-rm.sh

INPUT=$(cat)
COMMAND=$(echo "$INPUT" | python3 -c "import json,sys; d=json.load(sys.stdin); print(d.get('command',''))")

if echo "$COMMAND" | grep -qE "rm -rf (/|~|\.\./)"; then
    echo '{"decision":"block","reason":"rm -rf targeting root, home, or parent directory is blocked."}'
    exit 0
fi

echo '{"decision":"approve"}'
```

settings.json 配置：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/block-dangerous-rm.sh"
          }
        ]
      }
    ]
  }
}
```

Claude Code 尝试匹配危险 rm 模式的 Bash 时，钩子触发，返回阻止决策，命令不执行。

### Claude Code 如何处理阻止事件

PreToolUse 钩子阻止工具调用时，Claude Code 不停止。它收阻止原因，纳入上下文，通常：

1. 重新考虑方法，试替代方案
2. 问用户不同方法是否可接受
3. 报告无法在约束下完成

此为正确行为。钩子非崩溃器而是重定向器。设计良好的钩子在原因字符串中提供足够信息帮 Claude Code 找替代路径。

### 与 OpenTalon 的关联

OpenTalon 中，PreToolUse 钩子强制执行开发中最重要两约束：危险文件系统操作（此节）和写入代码的秘密（第 6.5 节）。两约束都可在 CLAUDE.md 中作为规则——但依赖注意力的规则弱于依赖结构性执行的钩子。钩子每匹配工具调用触发，每次，无一例外。第 6.5 节配置完整钩子系统时，两钩子同时就位，每会话从首工具调用受保护。
