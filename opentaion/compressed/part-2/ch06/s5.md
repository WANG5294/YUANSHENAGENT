## 第 6.5 节："代码中禁止 API 密钥"钩子

保存时格式化和通知钩子改善工作流质量。本节构建保护安全的钩子。Claude Code 有时在源文件中直接写 API 密钥、令牌或凭据——非恶意，而是因它试图生成可工作代码，而所用配置值可能来自对话上下文字面量。被提交的秘密可使 API 密钥失效、导致意外费用、或暴露服务于未授权访问。

此钩子在发生前拦截。

### 为何仅规则不够

OpenTalon 的 CLAUDE.md 说："永不提交数据库连接字符串——用环境变量。""重要"标记，Claude Code 大多数情况遵循。但"大多数情况"对秘密不够。会话第三小时，上下文压力大，新集成测试可能为方便含硬编码测试令牌。开发者注意不到就提交，秘密进 git 历史。

Write 和 Edit 上的 PreToolUse 钩子可在文件被写前捕获。检查针对即将写的内容，非已有文件。阻止在 git 看到内容前发生。

### 完整钩子脚本

```bash
#!/bin/bash
# .claude/hooks/no-api-keys.sh

INPUT=$(cat)

CONTENT=$(echo "$INPUT" | python3 -c "
import json, sys
d = json.load(sys.stdin)
content = d.get('content') or d.get('new_string') or ''
print(content)
" 2>/dev/null)

# OpenRouter / OpenAI keys: sk- + 40+ alphanumeric
if echo "$CONTENT" | grep -qE 'sk-[A-Za-z0-9]{40,}'; then
    echo '{"decision":"block","reason":"File contains what appears to be an OpenAI/OpenRouter API key (sk-...). Use environment variables instead: os.environ[\"OPENROUTER_API_KEY\"]"}'
    exit 0
fi

# Bearer tokens: Bearer + 30+ chars
if echo "$CONTENT" | grep -qE '"Bearer [A-Za-z0-9+/=_-]{30,}"'; then
    echo '{"decision":"block","reason":"File contains a hard-coded Bearer token. Use environment variables instead."}'
    exit 0
fi

# Supabase keys: eyJ... (base64 JWT)
if echo "$CONTENT" | grep -qE '"eyJ[A-Za-z0-9+/=]{50,}"'; then
    echo '{"decision":"block","reason":"File contains what appears to be a Supabase JWT key. Use environment variables: os.environ[\"SUPABASE_ANON_KEY\"]"}'
    exit 0
fi

# Generic key-like variable assignments
if echo "$CONTENT" | grep -qiE '(api_key|secret_key|auth_token|access_token)\s*=\s*"[A-Za-z0-9+/=_-]{32,}"'; then
    echo '{"decision":"block","reason":"File contains a hard-coded credential (api_key, secret_key, or token assigned a literal value). Use environment variables instead."}'
    exit 0
fi

echo '{"decision":"approve"}'
```

### 刚才发生了什么

三钩子脚本现在协同工作。每次文件写入时：

1. no-api-keys PreToolUse 先触发，扫描写入内容
2. 安全则写入继续
3. format-python PostToolUse 触发，格式化 Python 文件

no-api-keys 触发则写入被阻止，Claude Code 收含正确替代方案的具体错误信息。开发者从不需要考虑 Claude Code 是否记"禁止秘密"规则。

### 模式校准

钩子中模式故意保守。过激进会阻止合法写入——测试夹具占位符、注释示例代码、超 32 字符的 UUID。误报比要解决的问题更糟，因中断合法工作并训练开发者绕过。

钩子在合法内容上触发时，加例外：用 `# SAFE: not a real key` 注释检测到的行，更新检测模式跳过。像维护代码一样维护钩子——模式变化需更新。

### 与 OpenTalon 的关联

OpenTalon 中，此钩子是秘密到 git 历史前的最后防线。CLAUDE.md 规则是第一道。`.env` 模式和 `.gitignore` 是第二道。此钩子是第三道——即使开发者匆忙、Claude Code 上下文饱和、会话运行三小时注意力下降，它也运行。结构上运行，每次。

### 里程碑 M5：自动化质量门禁已激活

完整钩子系统就位——危险命令阻止、保存时格式化、OS 通知、API 密钥检测——里程碑 M5 完成。每次写入被扫描，每次 Python 编辑自动格式化，每次完成触发通知。开发环境结构性强制执行质量。
