## 第 6.3 节：PostToolUse 钩子：自动化代码质量

上节介绍强制执行：事前阻止。PostToolUse 钩子在工具调用成功后运行。它们不阻止——做出反应。典型用例是代码质量自动化：Python 文件被写或编辑后，自动运行格式化器和 linter。文件已变；钩子在 Claude Code 继续前清理。

无钩子时，会话依赖 Claude Code 记得文件编辑后运行格式化。有时记得，有时上下文压力导致跳过，有时格式化了但用错工具。PostToolUse 钩子使格式化结构性——总是发生，通过钩子，无论 Claude Code 决定什么。

### 保存时格式化模式

我们将建的钩子在任何被写或编辑的 Python 文件上运行 `ruff check --fix` 和 `black`。`ruff` 是代码检查器（快，与 pyproject.toml 兼容）。`black` 是格式化器。两者顺序运行：ruff 先修 lint，black 格式化结果。

钩子脚本通过 stdin 以 JSON 收工具调用输出，提取文件路径，运行质量工具：

```bash
#!/bin/bash
# .claude/hooks/format-python.sh

INPUT=$(cat)
FILE_PATH=$(echo "$INPUT" | python3 -c "
import json, sys
d = json.load(sys.stdin)
print(d.get('filePath') or d.get('path') or '')
")

if [[ "$FILE_PATH" != *.py ]]; then
    exit 0
fi

cd "$(git rev-parse --show-toplevel 2>/dev/null || pwd)"

uv run ruff check --fix "$FILE_PATH" 2>&1 | grep -v "^Found\|^All checks"
uv run black "$FILE_PATH" --quiet 2>&1

exit 0
```

settings.json：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/format-python.sh"
          }
        ]
      }
    ]
  }
}
```

### 钩子接收的内容

PostToolUse 钩子收 JSON 负载，含工具名称、输入（传给工具的内容）和输出（结果）。Write 工具相关字段是 `filePath`，Edit 是 `path`。钩子提取存在的字段。

路径提取用小的 Python 一行命令解析 stdin JSON。钩子脚本中常见模式：负载 JSON，Python 可靠解析，脚本用提取值继续。Bash 字符串解析 JSON 脆弱；非平凡提取用 Python。

### PostToolUseFailure 变体

`PostToolUseFailure` 在工具调用失败时触发——Bash 返回非零、文件写因权限错误失败、或其他工具调用遇错。钩子收错误信息。

OpenTalon 中实际用例是日志记录：测试运行失败时记下哪个失败、何时、错误是什么。创建持久错误日志超越会话上下文存在。Claude Code 可调试会话开始时读日志了解什么一直失败及时间。具体实现属第 10 章。

### 为何比要求 Claude Code 格式化更好

每 CLAUDE.md 写"编辑 Python 后总运行 black 和 ruff"的方案——Claude Code 以中等稳定性遵循。80% 准确率下，20% 文件编辑留未格式化代码。这些累积。代码审查 diff 含无关格式化噪音。CI lint 失败。开发者手动清理。

100% 准确率——钩子提供的——这些都不会发生。钩子不被复杂 bug 修复分心忘格式化。每匹配工具调用运行，每次。代价是每次文件写不足一秒和零开发者注意力。

### 与 OpenTalon 的关联

OpenTalon 中，此钩子是 CLI 和 API 组件的自动质量层。Claude Code 在 `api/src/opentaion_api/routers/proxy.py` 写新路由处理器时，钩子立即触发，ruff 捕获导入顺序或未用变量问题，black 规范化格式。仓库中累积代码从首次提交就是一致格式化的。第 17 章 CI 流水线运行自己 lint 时将通过——非因记得格式化，而是因钩子让忘变不可能。
