## 第 6.1 节：12 个生命周期事件

技能规定 Claude Code 显式调命令时的行为。钩子则规定 Claude Code 在代理循环特定节点自动执行的行为，无需显式调用。它们是技能层之下的自动化层——强制执行约束和运行质量检查，无需人类记得去要求。

Claude Code 暴露十二个生命周期事件，每事件对应执行周期中一特定时刻。理解各事件触发时机，是知哪些钩子该放哪的前提。

### 十二个事件

**会话事件（Session events）** 会话开始或结束时触发：

- `SessionStart` —— Claude Code 会话打开时触发。设环境变量、检查工具是否可用、或加载会话所需外部上下文。
- `SessionEnd` —— 会话关闭时触发。实践中少用；会话可能突然结束（终端关、超时），事件非所有情况下可靠触发。
- `Setup` —— 处理首条用户消息前触发。与 SessionStart 类似，但稍晚，在 CLAUDE.md 加载完成之后。

**工具事件（Tool events）** 单工具调用前后触发：

- `PreToolUse` —— 工具调用执行前触发。钩子接收工具名称和输入，可批准、阻止或修改调用。强制执行点：若阻止，工具不执行。
- `PostToolUse` —— 工具调用成功完成后触发。接收名称、输入和输出。自动化质量检查：文件写入后 lint，Bash 后记日志。
- `PostToolUseFailure` —— 工具调用失败时触发。接收错误信息。错误日志或告警。

**代理事件（Agent events）** 子代理执行时触发（多代理工作流）：

- `SubagentStart` —— 生成子代理开始时触发。
- `SubagentStop` —— 子代理完成或终止时触发。

**用户事件（User events）** 用户交互时触发：

- `UserPromptSubmit` —— 用户提交消息时，Claude Code 处理前触发。可向每条消息注入额外上下文。
- `PermissionRequest` —— Claude Code 请求用户授权时触发。钩子可自动批准或拒绝特定类别请求。

**完成事件（Completion events）** 工作完成或代理需关注时触发：

- `Stop` —— 代理循环终止时触发（任务完成或进入错误状态）。
- `Notification` —— Claude Code 想提醒用户时触发——通常长任务完成且代理等待输入时。

### 实际常用的子集

十二事件完整列表。实践中，四事件覆盖绝大多数用例：

**`PreToolUse`** —— 强制执行钩子。危险操作执行前拦截。检测 API 密钥的钩子放这。阻止特定目录外 `rm -rf` 的钩子放这。结构性约束都应放 PreToolUse 钩子。

**`PostToolUse`** —— 质量自动化钩子。每写 Python 文件后运行 black 和 ruff。每次测试运行后检查结果记日志。操作后自动执行的质量检查放这。

**`Notification`** —— 开发者生产力钩子。Claude Code 完成长任务等待输入时触发 OS 通知，你可关终端窗口切其他应用。此钩子将 Claude Code 从需盯着的终端变后台工作者。

**`SessionStart`** —— 环境设置钩子。检查 CLI 工具是否安装，验证环境变量，运行首任务前任何初始化工作。

### 钩子的配置方式

钩子在 `.claude/settings.json`（项目级，git 共享）或 `~/.claude/settings.json`（用户级）中配置。结构将生命周期事件映射到钩子数组，每钩子指定匹配器（matcher）和动作（action）：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/format-python.sh"
          }
        ]
      }
    ]
  }
}
```

`matcher` 指定哪些工具调用触发钩子。`type` 指定动作，最常见 `command` —— shell 脚本，通过 stdin 接收工具调用上下文，返回决策或输出。

安全提示：钩子脚本会话启动时快照。若会话中途修改钩子脚本，修改不生效直到会话重启。行为与预期不符时用 `/hooks` 查看活动钩子。

### 与 OpenTalon 的关联

OpenTalon 中，钩子系统是自动化质量基础设施最后拼图。第 5 章赋予 Claude Code 搭代码框架的技能。第 6 章赋予它自动强制执行代码质量的约束。第 6.3 和 6.5 节写保存时格式化钩子和 API 密钥检测钩子后，项目每次文件写入都将自动格式化和秘密扫描——无需显式提醒或手动步骤。开发者工作变指定意图；钩子负责正确性。
