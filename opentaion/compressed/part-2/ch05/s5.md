## 章节 5.5：/effort 命令：调整思考深度

前两节两技能已就位：`/opentaion-component` 和 `/api-endpoint` 在 `.claude/skills/` 中，可供第三部分构建用。结束第 5 章前，还有第三工具对每会话重要：`/effort` 命令，控制 Claude Code 应用于每个任务的推理量。

扩展思考（extended thinking）非二进制开关。Claude Code 支持的模型可在生成响应前执行不同量内部推理——推理对用户不可见，但效果不是。最小思考和密集思考的任务可产生实质不同输出。`/effort` 让你有意识将每任务放在此频谱上。

### /effort 控制什么

`/effort` 设思考 Token 预算：Claude Code 在生成响应前可用于内部推理的最大 Token 数。预算在推理中消耗；输出 Token 在其外。

四级别：

**`/effort low`**——约 4,000 思考 Token。快速。适用样板生成、格式化修复、重命名等输出由机械方式决定的任务。有明确正确答案的任务不需扩展推理。

**`/effort medium`**——约 10,000 思考 Token。多数实现工作的平衡默认。适用按规格实现功能、为现有代码写测试、或有明确诊断的针对性 Bug 修复。

**`/effort high`**——约 20,000 思考 Token。适用根因不明显的调试、涉及多关联文件的重构、或智能体需考虑几种替代方案后才决定采用哪个的任务。

**`/effort max`**——约 31,000 思考 Token。保留给难逆转且错误后果严重的设计决策：写 SPEC.md、设计 API 模式、做影响数十个后续决策的架构选择。成本是低思考水平的 3-5 倍。

### 默认行为

Claude Code 在支持扩展思考的模型上默认用它。默认约中等——足够推理处理多数任务，又不至每会话在日常工作上耗尽预算。`/effort` 从默认值上下调。你无需每会话设它；只在知特定任务需不同级别时设。

### OpenTalon 中何时用每级别

OpenTalon 开发中，每级别在构建序列中有其自然位置：

**低**——运行 `/opentaion-component display` 搭新文件。结构由技能固定，不需推理改变输出。脚手架前设 `/effort low`。

**中**——第 9 章实现 `run_loop()`。规格存在（第 9.5 节 SPEC.md），测试存在（第 10 章），任务是将测试变绿。从清晰规格实现是中等。

**高**——第 12 章调试上下文溢出 Bug，智能体 40 轮后失连贯。根因不立即明显，需测几个假设，修复可能不平凡。高努力合适。

**最高**——第 9.3 节写 OpenTalon SPEC.md，或第 15.4 节设计代理端点模式。这些将被本书其余部分引用。第一次做对成本低于在其上构建二十个下游章后修改的成本。

### 成本含义

最高思考的 3-5 倍成本乘数非回避理由——而是有意使用理由。架构决策时最高努力运行、脚手架任务切换到低努力的会话，将推理预算花在回报最高处。一切任务都保持最高努力的会话，在为常规实现支付架构推理价格，浪费。

`/cost` 命令实时显示当前会话 Token 消耗。用 `/effort max` 做 SPEC.md 后检查 `/cost`。数字有启发性，还将帮你校准何时最高思考合理，何时高级思考就够。
