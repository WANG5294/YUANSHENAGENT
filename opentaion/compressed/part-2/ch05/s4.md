## 章节 5.4：编写 /api-endpoint 技能

`/opentaion-component` 处理 CLI 脚手架。`/api-endpoint` 处理另一侧：给定路由器名称和 HTTP 方法，搭建带 Pydantic 请求响应模型、异步处理器和 pytest 测试的 FastAPI 路由。模式与上节相同——先读后写，生成脚手架不生成实现，让用户处清晰起点——应用于不同领域。

一设计决策使此技能与组件技能区分：它无 Bash 访问。组件技能需 Bash 运行测试并确认导入成功。API 端点技能只读写文件。运行迁移、启服务器或运行集成测试是有副作用的操作，应有意为之，非脚手架时自动完成。`allowed-tools` 列表使约束显式。

### 技能做什么

给定像 `/api-endpoint usage GET` 的调用，该技能：

1. 读 `api/src/opentaion_api/routers/` 中现有路由器结构
2. 创建或追加到适当路由器文件，添遵循 FastAPI 约定的新路由
3. 为新端点创建 Pydantic 请求和响应模型
4. 在 `api/tests/` 创建测试存根，导入路由器并测试存在
5. 报告创建了什么，不运行任何东西

技能不启服务器、不运行迁移或测试。这些是开发者下一步。

### 完整的技能文件

在 `.claude/skills/api-endpoint/SKILL.md` 创建：

```markdown
---
name: api-endpoint
description: 用 Pydantic 模型和测试存根搭建 FastAPI 路由
argument-hint: <router-name> <HTTP-method>
allowed-tools:
  - Read
  - Glob
  - Write
---

# Skill: api-endpoint

为 OpenTalon API 搭建一个新的 FastAPI 路由。

## 参数

`$ARGUMENTS` — 两个词：路由器名称和 HTTP 方法
  示例：`usage GET`、`keys POST`、`proxy POST`

## 过程

1. **读取现有路由器结构**
   - 使用 Glob 查看 `api/src/opentaion_api/routers/*.py`
   - 读取与第一个参数匹配的路由器文件（如果存在）
   - 如果不存在，读取一个现有路由器以理解模式

2. **解析参数**
   - 路由器名称：`$ARGUMENTS` 的第一个词
   - HTTP 方法：`$ARGUMENTS` 的第二个词
   - 推导端点路径：`/{router-name}`

3. **创建或更新路由器文件**
   写入或追加到 `api/src/opentaion_api/routers/{router_name}.py`：

   ```python
   # api/src/opentaion_api/routers/{router_name}.py
   from fastapi import APIRouter, Depends
   from pydantic import BaseModel
   from ..auth import get_current_user

   router = APIRouter(prefix="/{router_name}", tags=["{router_name}"])

   class {RouterName}Response(BaseModel):
       # TODO: 定义响应字段
       pass

   @router.{method}("/")
   async def handle_{router_name}_{method_lower}(
       current_user = Depends(get_current_user),
   ) -> {RouterName}Response:
       """[此端点的简短描述]"""
       raise NotImplementedError("Endpoint not yet implemented")
   ```

4. **创建测试存根**
   写入 `api/tests/test_{router_name}.py`：

   ```python
   # api/tests/test_{router_name}.py
   from fastapi.testclient import TestClient
   from opentaion_api.main import app

   client = TestClient(app)

   def test_{router_name}_{method_lower}_exists():
       response = client.{method_lower}("/{router_name}/")
       assert response.status_code in [200, 401, 422]
   ```

5. **报告**
   告诉用户创建或修改的文件、脚手架路由、下一步做什么

## 备注
- `{RouterName}` 骆驼命名法版
- `{method_lower}` 小写 HTTP 方法
- 不运行服务器、测试或 Bash——由开发者运行
- 若路由器文件已存在，添新路由不碰现有
```

### 关键设计决策

`allowed-tools` 限制为 `[Read, Glob, Write]` 是有意约束。有 Bash 访问的脚手架技能可能决定运行 `alembic upgrade head` 或 `uvicorn main:app` 作为过程部分——数据库迁移和服务器重启，有真实后果，可能因与所搭路由无关原因失败，且开发者需决定何时运行。排除 Bash 后，影响范围限文件创建。开发者保留对涉及运行系统的一切控制。

此即第 3 章原则应用于技能设计：风险层级从 Read（无副作用）经 Write（创建文件）到 Bash（执行任意命令）。用任务所需最小访问级别。

### OpenTalon 连接

OpenTalon 中，`/api-endpoint` 是加速第 15-16 章（API 逐条路由构建）的工具。代理端点、用量端点、API 密钥管理端点都是一行调用加实现。无此技能，Claude Code 每次添加路由都需重新发明路由结构。有此技能，路由结构由脚手架固定，实现是唯一变量。技能尚未用——API 目录还不存在——但已就位等待构建开始。
