## 章节 5.3：编写 /opentaion-component 技能

前节描述技能是什么及 Claude Code 如何发现。本节构建一个。`/opentaion-component` 解决 CLI 开发中特定重复问题：添新命令时每次都做相同操作——正确目录建新文件、正确导入、正确 Click 装饰器定义命令、创建匹配测试存根。无技能时，Claude Code 每次不同方式执行，产生略微不一致结构，累积成维护问题。有技能，过程编码一次并一致应用。

### 技能做什么

给定组件名称——如 `/opentaion-component config`——该技能：

1. 读 `cli/src/opentaion/` 现有结构，理解当前模式
2. 创建 `cli/src/opentaion/config.py`，含正确导入、Click 命令定义和存根实现
3. 创建 `cli/tests/test_config.py`，含最小 pytest 测试，导入新命令并验证存在
4. 运行测试确认以预期方式失败（导入成功，行为未实现——TDD RED 状态）
5. 报告创建了什么及下一步

技能不实现功能。创建脚手架使实现成为下一自然任务。

### 完整的技能文件

在 `.claude/skills/opentaion-component/SKILL.md` 创建：

```markdown
---
name: opentaion-component
description: 用测试存根搭建新的 OpenTalon CLI 命令
argument-hint: <component-name>
allowed-tools:
  - Read
  - Glob
  - Write
  - Bash
---

# Skill: opentaion-component

为 OpenTalon 智能体搭建一个新的 CLI 命令。

## 参数

`$ARGUMENTS` — 蛇形命名法（snake_case）的组件名（例如 `config`、`usage_report`）

## 过程

1. **读取现有结构**
   - 使用 Glob 查看 `cli/src/opentaion/*.py` 以了解现有命令文件
   - 读取一个现有命令文件以理解导入和装饰器模式

2. **创建命令文件**
   写入 `cli/src/opentaion/{$ARGUMENTS}.py`：

   ```python
   # cli/src/opentaion/{component_name}.py
   import click
   from rich.console import Console

   console = Console()

   @click.command()
   @click.argument("input", required=False)
   def {component_name}(input: str | None) -> None:
       """[此命令功能的简短描述]"""
       console.print(f"[yellow]{component_name}:[/yellow] not yet implemented")
   ```

3. **创建测试存根**
   写入 `cli/tests/test_{$ARGUMENTS}.py`：

   ```python
   # cli/tests/test_{component_name}.py
   from opentaion.{component_name} import {component_name}
   from click.testing import CliRunner

   def test_{component_name}_exists():
       runner = CliRunner()
       result = runner.invoke({component_name}, [])
       assert result.exit_code == 0
   ```

4. **运行测试**
   ```bash
   cd cli && uv run pytest tests/test_{$ARGUMENTS}.py -v
   ```
   确认：导入成功且测试通过（存根是功能性的）。

5. **报告**
   告诉用户：
   - 创建了哪些文件
   - 测试结果
   - 下一步：实现 `{component_name}()` 以替换存根

## 备注
- 将全文中的 `{component_name}` 替换为实际的 `$ARGUMENTS` 值
- 不要实现功能——只搭建结构
- 如果 `$ARGUMENTS` 包含连字符，转换为下划线以作为 Python 标识符
```

### 为什么这个技能存在

考虑无此技能会发生什么：每次让 Claude Code"向 CLI 添加 config 命令"，它都重新决定如何结构化文件、用什么导入、是否创建测试、测试放哪、测试验证什么。输出各不相同。十个命令后，代码库有十种略微不同风格——有些有测试有些没，有些导入了 Rich 有些没。

该技能一次性编码这些决策。读取步骤（步骤 1）意味着技能始终匹配代码库已有任何模式，即使模式演变。若第 9 章改变命令结构方式，技能下次调用时自动采用新结构，因写前先读。

### `$ARGUMENTS` 变量

用户调 `/opentaion-component config` 时，字符串 `config` 在技能内作为 `$ARGUMENTS` 可用。技能可在散文指令中引用此变量，Claude Code 解释并替换。此即技能如何参数化：YAML `argument-hint` 声明期望形式，`$ARGUMENTS` 接收实际调用值。

### OpenTalon 连接

OpenTalon 中，此技能是从第 9 章开始实现 CLI 命令时将用的组件创建工作流首部分。它还没做任何可见的事——CLI 目录还不存在。但技能文件本身在 `.claude/skills/opentaion-component/SKILL.md`，CLI 实现开始时它在那。来自第 2 章原则的应用：工具先于工作。现在写技能意味着到实现章节时，脚手架过程已定义好且一致。
