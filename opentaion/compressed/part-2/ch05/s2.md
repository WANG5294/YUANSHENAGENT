## 章节 5.2：技能系统：YAML 前置元数据和作用域

内置命令覆盖会话管理、模型控制和工作流原语。它们不可扩展。当你需对项目做特定事情的命令——正确位置搭新 CLI 命令、遵循正确模式、运行正确测试——那就是技能（skill）。

技能是有 YAML 前置元数据的 Markdown 文件，Claude Code 发现并作为斜杠命令提供。机制不复杂。唯一最重要实现细节是目录结构——写代码前精确理解。

### 能工作的结构

名为 `my-skill` 的技能须创建于：

```
.claude/skills/my-skill/SKILL.md
```

即 `.claude/skills/` 内以技能命名的子目录，含精确命名为 `SKILL.md` 的文件。Claude Code 启动扫描技能时专门找此模式。技能以 `/my-skill` 形式可用。

### 静默失败的结构

以下路径看似合理：

```
.claude/skills/my-skill.md
```

调用时产生"Unknown skill: my-skill"错误，无额外解释。启动无错误，无文件被发现但格式错误的警告。文件存在，Claude Code 忽略它，命令什么也不做。此最常见技能设置错误，因直到调用时才无声失败，极难诊断。

遇"Unknown skill"错误且确信已创建技能，先检查目录结构。带内部 `SKILL.md` 的子目录非可选。

支持性文件的好处是接受此结构的理由：因技能存在目录而非单文件中，可在 `SKILL.md` 旁包含模板、参考文档、示例输入和脚本。技能执行期间可用相对路径读取。脚手架技能可将文件模板包含在自己目录中，非嵌入为 SKILL.md heredoc 字符串。

### YAML 前置元数据

`SKILL.md` 以 `---` 间 YAML 开始：

```yaml
---
name: opentaion-component
description: 按照 OpenTalon 约定搭建一个新的 CLI 命令及测试
argument-hint: <component-name>
allowed-tools:
  - Read
  - Write
  - Bash
---
```

**`name`** — 技能标识符。须与目录名匹配。Tab 补全显示及 `/` 后输入的内容。

**`description`** — 帮助显示和 Tab 补全预览中显示。此字符串也消耗上下文预算——所有已发现技能描述约占用 2%。保持描述二十字内。"搭建新 CLI 命令及测试"好。"此技能帮你按所有已建立模式在 OpenTalon 搭新 CLI 组件包括测试文件"则非。

**`argument-hint`** — Tab 补全中显示的参数语法。尖括号必选（`<component-name>`），方括号可选（`[--dry-run]`）。只是文档——技能主体决定对参数做什么。

**`allowed-tools`** — 明确限制技能可用工具。省略则继承会话完整工具集。安全性和可预测性控制：只应读写文件的脚手架技能应声明 `[Read, Write]` 非 `[Bash]`，不可意外运行 Shell 命令。第 8 章安全角度再讨论。

**`model`** — 可选。覆盖会话模型。技能需比默认更重推理（`claude-opus-4-6`）或更快执行（`claude-haiku-4-5-20251001`）时用。多数技能不应设，继承会话。

### 三个作用域级别

技能从三位置发现：

**项目作用域** — 项目根 `.claude/skills/`。这些提交到仓库共享（OpenTalon 跨越所有写书会话）。你正用的 `write-section`、`session-wrap`、`autonomous-write` 即项目作用域。

**用户作用域** — `~/.claude/skills/`。个人化——每项目可用，不通过 Git 共享。通用脚手架技能放此。

**子目录作用域** — 单仓库子目录内 `.claude/skills/`（如 `cli/.claude/skills/`）。仅当 Claude Code 在子目录内工作时可用。对太组件特定、不同组件时不应出现在 Tab 补全中的技能有用。

Claude Code 启动扫描所有三级。同名存在于多级则最具体者胜（子目录 > 项目 > 用户）。

### 旧版格式

Claude Code 也支持 `.claude/commands/` 中扁平的 Markdown 技能。原始格式。两种都可用。若为 OpenTalon 用 BMAD V6（第四部分介绍），其智能体定义用 `.claude/commands/` 格式——没问题，有意的。`.claude/skills/<name>/SKILL.md` 子目录格式是现代方法。

### OpenTalon 连接

OpenTalon 中，三现有技能（`write-section`、`session-wrap`、`autonomous-write`）展示上述原则。位于 `.claude/skills/<name>/SKILL.md`，有精确描述。`autonomous-write` 故意不含 `allowed-tools` 限制，因需完整工具集写章节、读文件、更新进度。我们将写的两技能——`/opentaion-component` 和 `/api-endpoint`——用显式 `allowed-tools` 限制，因范围更窄。设计有意：为任务用最小工具集并明确声明使约束可见。
