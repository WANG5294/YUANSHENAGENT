## 章节 5.1：内置命令：完整参考

Claude Code 自带一组斜杠命令，从首次按键即可用。无需配置设置。它们作用于会话、模型、工作流和项目配置。部分在 Tab 补全中可见；少数不显眼但对开发重要。本节是参考：每命令做什么及何时用。

开始前澄清：这些与技能不同。内置命令是 Claude Code 本身一部分——无论你在哪项目、有什么 `.claude/` 目录，它们都存在。技能（第 5.2 节介绍）是你自己写的项目特定命令。内置命令不可定制；技能可以。

### 会话管理

**`/clear`** — 清空对话历史重新开始。之后上下文窗口为空。CLAUDE.md 文件在下条消息时重载。不相关任务间用，或想开始新任务而不带先前上下文累积时用。代价是丢失会话中已建立的一切——仅代价可接受时用 `/clear`。

**`/compact`** — 将对话历史压缩为摘要，释放上下文空间而不完全丢失工作。Claude Code 写已完成工作摘要，然后从摘要继续。上下文中途填满且需继续同一任务不从头开始时用。摘要可能丢失细节——重要上下文应放文件非仅对话。

**`/cost`** — 显示当前会话 Token 数和预估成本。对校准成本感知有用。见一次大型调试消耗 40K Token 后，未来同类任务成本变得可预测。

**`/status`** — 显示当前模型、思考模式、权限设置等。确认所期望的运行确实在运行，尤其切换模型或调整权限后。

### 模型和推理控制

**`/model`** — 会话中切换活动模型。示例：`/model claude-opus-4-6` 切 Opus 处理复杂架构决策，再切回 Sonnet 实现。会话中切换保留对话上下文——模型更改在下条消息时生效。

**`/effort`** — 调整思考 Token 预算。第 5.5 节详述。简言之：控制 Claude Code 回复前内部推理量。`/effort low` 快且便宜；`/effort max` 彻底但昂贵。第 5.5 节涵盖 OpenTalon 开发中每级别何时适用。

### 工作流命令

**`/review`** — 触发代码审查。Claude Code 读差异，应用 CLAUDE.md 中编码约定，生成带改进建议的结构化审查。提交多文件更改前用。

**`/pr_comments`** — 读当前 PR 上公开评论（GitHub 连接工作流中）并处理。PR 有多审阅者评论且希望按顺序处理时有用。

**`/release-notes`** — 从上标签以来提交历史生成发布说明。总结为用户可读语言，按类型分组，生成可编辑草稿。

### 配置和内省

**`/memory`** — 打开 CLAUDE.md 层级编辑。显示每级别（组织、用户、项目、子目录）活动记忆文件，允许添加或编辑。想会话中临时加规则时用，非手动导航到 `~/.claude/CLAUDE.md` 或项目文件。

**`/permissions`** — 显示当前权限设置并允许调整。权限模式（自动批准、每次询问等）在第 8 章介绍。临时提升或限制权限而不重启会话时用 `/permissions`。

**`/mcp`** — 列出连接的 MCP 服务器及其可用工具。验证 GitHub、PostgreSQL 或 Playwright MCP 服务器是否连接并暴露出期望工具。第 7 章构建完整 OpenTalon MCP 栈。

**`/hooks`** — 列出活动生命周期钩子。显示哪些事件注册钩子、运行什么命令、阻塞还是非阻塞。诊断钩子为何触发或没触发。第 6 章覆盖完整钩子系统。

### 实用工具

**`/doctor`** — 对 Claude Code 安装运行诊断：检查所需二进制、验证 `.claude/` 目录结构、测试 MCP 连接、报告问题。东西不按预期工作时运行。

**`/init`** — 扫描当前仓库据发现生成起步 CLAUDE.md：package.json 识别前端栈，pyproject.toml 识别 Python 栈，配置提示 linting 和测试约定。结果需编辑的草稿——但对无 CLAUDE.md 的现有项目，比从头写快。OpenTalon 在代码存在前就写了 CLAUDE.md，故 `/init` 在此无用。对棕地项目，它是起点。

**`/vim`** — 切换 Vim 键绑定。个人偏好，无功能影响。

**`/terminal-setup`** — 配置终端集成：设 Shell 补全、配置提示集成、启用终端级功能。新机器上运行一次。

### 对 OpenTalon 最重要的三个命令

OpenTalon 开发中，三命令将在剩余章节反复出现：

- **`/compact`** —— 长构建会话填满上下文需继续不从头时用
- **`/clear`** —— 主要里程碑间用，开始新阶段不带上下文噪音
- **`/effort`** —— 为每种任务类型调推理深度：脚手架低，实现中，调试高，架构最高

其余知道就好，少在任务中间用。下节从内置命令转向让你构建自己命令的系统。
