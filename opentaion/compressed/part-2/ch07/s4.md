## 第 7.4 节：Playwright MCP：测试 Web 仪表板

GitHub MCP 服务器覆盖了代码审查。PostgreSQL MCP 服务器覆盖了数据。Playwright MCP 服务器覆盖了两者都无法触及的一个领域：渲染后的 Web 界面。当 OpenTalon Web 仪表板上线后，测试它需要一个浏览器。没有 Playwright MCP，即开发者需要手动点击注册流程、API 密钥管理界面和使用量图表——每次更改时都通过肉眼验证每一次交互。

Playwright MCP 让 Claude Code 直接做这些测试，打开一个浏览器，导航到页面，填写表单，点击按钮，截取屏幕截图，并读取页面结构。UI 测试的反馈循环在代理会话内部闭合。

### Playwright MCP 提供的能力

Playwright MCP 服务器将浏览器自动化暴露为一组工具：

**导航** ——`navigate(url)``go_back()``reload()`。Claude Code 可打开任何 URL，包括开发期间的`localhost:5173`。

**交互** ——`click(selector)``fill(selector, value)``press(key)``select_option(selector, value)`。点击按钮、填写表单输入、按快捷键。

**检查** ——`get_page_snapshot()`返回当前页面的无障碍树（accessibility-tree）表示，以结构化文本形式呈现。Claude Code 阅读此描述以理解页面结构，无需以视觉方式解释屏幕截图。`screenshot()`需要视觉检查时捕获图像。

**断言** ——`find_element(selector)`检查元素是否存在，`get_text(selector)`从元素中读取文本内容。

无障碍快照功能值得关注。Playwright MCP 不是要求 Claude Code 以视觉方式解释屏幕截图（这既慢又可能遗漏细节），而是将页面表示为文本树：按钮、标题、表单输入、链接——每个都有其角色、文本内容和相关属性。Claude Code 直接推理这个文本结构，测试逻辑中不需要视觉能力。

### 配置

Playwright MCP 作为本地 stdio 服务器运行，会启动一个真正的浏览器进程：

```bash
# Install Playwright MCP and the browser dependencies
npm install -g @playwright/mcp
npx playwright install chromium

# Add to .mcp.json (project-scoped — all teammates share it)
```

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp", "--browser", "chromium"],
      "type": "stdio"
    }
  }
}
```

不需要 API 令牌。该服务器在本地生成一个 Chromium 进程。浏览器在 Claude Code 首次使用 Playwright 工具时打开，会话结束时关闭。

### 一个具体的测试示例

当 Web 仪表板在`localhost:5173`上运行时，像这样的任务变得可能：

```
测试魔法链接注册流程：
1. 打开仪表板登录页面
2. 输入电子邮件地址 test@example.com
3. 点击"发送魔法链接"按钮
4. 验证成功消息出现
5. 报告流程是否按预期工作
```

Claude Code 使用 Playwright MCP 执行每一步：`navigate("http://localhost:5173")``get_page_snapshot()`以了解表单结构、`fill("#email-input", "test@example.com")``click("button[type=submit]")`然后再次`get_page_snapshot()`以验证成功消息是否存在。

这并不是第 18 章中构建的自动化 E2E 测试套件的替代品。Playwright MCP 测试是探索性和交互式的——它们开发中验证行为。第 18 章中的 pytest-playwright 测试是 CI 中运行的回归测试。两者各有其位；它们服务于反馈循环的不同部分。

### 与 OpenTalon 的关联

OpenTalon 中，Playwright MCP 是将 Claude Code 的编码工作与其产生的用户体验连接起来的工具。当第 16 章中对`Dashboard.tsx`的更改完成时，Claude Code 可以在 Playwright 控制的浏览器中打开页面，导航到使用量图表，并验证图表是否正确渲染——甚至开发者手动打开浏览器前。无障碍快照让 Claude Code 读取渲染后的组件树，并验证预期的数据存在。配置位于`.mcp.json`中，一旦开发服务器运行即可使用。

最后一个 MCP 部分解释了如何三个服务器及其组合工具集时管理，而不消耗整个上下文预算。
