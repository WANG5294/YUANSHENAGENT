## 第 7.5 节：工具搜索：防止上下文溢出

现在已配置了三个 MCP 服务器：GitHub、PostgreSQL 和 Playwright。每个服务器都暴露了数十个工具。其组合工具定义——名称、描述、JSON 参数 schema——任何对话开始前就可能消耗 40,000 到 70,000 个令牌。一个 200,000 令牌的上下文窗口中，这相当于第一个提示前就占用了 20-35%。

此即工具过载问题。它与第 3 章中描述的复合上下文成本相互叠加。一个会话以 70,000 令牌的工具定义开始，经过二十轮对话累积了 50,000 令牌的对话历史，然后须读取一个 5,000 令牌的文件作为上下文——其有效工作空间只有 75,000 令牌，从 200,000 令牌降了下来。推理质量随着上下文压力而下降。浪费在累积。

工具搜索（Tool Search）是 Claude Code 的解决方案。

### 工具搜索的工作原理

当工具定义超过可用上下文窗口的可配置阈值时，Claude Code 从加载所有工具定义切换到对它们进行语义搜索。Claude Code 不再每次轮次开始时接收所有 80 个工具定义，而是只接收与当前任务相关的工具的定义。

其机制是：启动时，Claude Code 索引所有可用的工具定义。每次轮次中，推理前，它使用当前任务描述作为查询对工具索引运行语义搜索。与任务最相关的 5-10 个工具被加载到上下文中。其余的不加载。若 Claude Code 需要一个它最初未加载的工具，它可以用更具体的查询再次搜索。

来自必读材料的数字是具体的：一个包含 80 多个工具的三服务器设置，没有工具搜索时可能加载约 72,000 令牌的定义。启用工具搜索后，同一个会话加载约 8,700 令牌——减少了 85%。除了节省上下文之外，Opus 4 的工具选择准确率从 49% 提高到 74%，当它从一个小的、语义相关的子集而非所有可用工具中选择时。

准确率的提升与令牌节省同样重要。更少的不相关工具意味着更少的错误工具调用、更少的不必要往返次数，及更快地收敛到正确的解决方案。

### 配置

当工具定义超过上下文窗口的 10% 时，工具搜索会自动激活。你可以降低阈值以更早触发：

```bash
# In your shell environment or .env
ENABLE_TOOL_SEARCH=auto:5
```

`auto:5`值工具定义超过上下文窗口的 5% 时触发工具搜索——比默认值更早，这对于有三个活动 MCP 服务器的设置是合适的。

OpenTalon 开发中，工具搜索是三个 MCP 服务器是资产还是负债的区别所在。没有它，GitHub + PostgreSQL + Playwright 的组合工具定义会每次会话中消耗相当大比例的上下文，无论这些工具在该会话中是否需要。有了它，上下文只工具与当前任务相关时才用于工具。配置就是一个环境变量；收益从第 7 章开始的每次会话中持续存在。

### 里程碑 M6：MCP 栈已配置

随着工具搜索配置完成，MCP 栈已完成。Claude Code 会话中验证配置：

```
/mcp
```

期望的输出：

```
MCP Servers:
  github (user-level): connected | 42 tools
  postgres (project):  connected | 8 tools
  playwright (project): connected | 18 tools

Total tools: 68
Tool Search: active (threshold: 5% of context)
```

若有任何服务器显示为"disconnected"，请检查：
1.`.env`中的连接字符串或令牌
2. 服务器二进制文件是否已安装（`npx @github/mcp --version`等）
3.`DATABASE_URL`是否已设置（对于 postgres，尚未连接——这是预期情况，直到第 15 章数据库配置好为止）

### 刚才发生了什么

第 7 章为 Claude Code 的工具箱增加了三个外部感知系统。GitHub MCP 连接了代码审查循环。PostgreSQL MCP 连接了数据层。Playwright MCP 连接了渲染后的 UI。工具搜索确保同时连接三个工具不会工作开始前就消耗完上下文预算。

这三个服务器不会都第四部分前被积极使用，那时 OpenTalon 正在构建。但它们已配置好且可用。当 Claude Code 第 17 章须读取一个 PR 评论时，连接已存在。当它需要第 16 章查询数据库时，连接字符串已在`.env`中等待数据库的存在。基础设施先于其支持的工作。

第 8 章将回到本地环境——不是为增加更多能力，而是为约束已存在的东西。
