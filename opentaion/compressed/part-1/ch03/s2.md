## 章节 3.2：Claude Code 的原生工具

六个工具覆盖 Claude Code 在开发会话中大部分工作。理解每个——用途、成本和风险——是良好使用它们的先决条件。

**Read**——将文件内容加载到上下文窗口。默认从头读最多 2,000 行。大文件可用偏移量（offset）和限制（limit）分段读。Read 是用于代码、配置和文档的主要感知工具，无副作用。

**Write**——创建新文件或完全替换现有文件。Write 生硬——替换目标文件完整内容。对不存在文件，它是正确选择。对已存在且只需部分更新的文件，它是危险选择：丢弃文件所有内容用新内容替换。对现有文件的 Write 会破坏之前存在的任何上下文。

**Edit**——对现有文件手术式更改：将特定字符串替换为新字符串，其他不变。对现有文件，Edit 优于 Write，因它精确显示改了什么、保留了未改内容、若找不到目标字符串会安全失败。Claude Code 编辑你想审查的文件时，你审查差异而非替换。文件长时此区别极重要。

**Bash**——在持久 Shell 会话中执行命令。工作目录在 Bash 调用间持久化；其他 Shell 状态（子 Shell 中设的环境变量、别名）不持久。Bash 最强大也最危险：可运行测试、编译代码、装包、Git 提交、删文件、调外部 API。使 `pytest` 有用的同一工具也使 `rm -rf` 可能。第 8 章关于如何将 Bash 限制为意图范围。

**Glob**——通过模式搜索文件。`**/*.py` 查找项目树中每个 Python 文件。结果按修改时间排序，用于发现仓库中存在文件而不读任何文件内容。快速、廉价、安全。

**Grep**——通过正则表达式搜索文件内容。`agent.run_loop` 查找含该函数调用的每个文件。Grep 是大型代码库导航工具——有针对性 Grep 结果通常仅 100 Token，而指向的完整文件可能有 5,000 个。需找相关代码时，先 Grep 后 Read，而非读所有代码。

风险层级从 Read（无副作用）经 Edit（改文件但保留上下文）和 Write（完全替换文件）到 Bash（执行任意命令）。设计良好的智能体任务会刻意按此层级推进：先观察（Read、Glob、Grep），再谨慎行动（Edit），最后只执行前序观察证明必要的操作（Bash）。

另两工具将感知范围扩展到本地文件系统外。**WebFetch** 检索 URL 内容并据提示总结。**WebSearch** 执行搜索查询并返回结果。这些允许 Claude Code 读文档、查库更新日志、或在本地代码库中无答案时查找。

**TodoWrite** 与其他不同：它不与文件系统或网络交互。它在智能体上下文中维护结构化任务列表——带状态字段（pending、in-progress、completed）的项目，Claude Code 可在工作时读写。对多步骤长时间任务，TodoWrite 是智能体跟踪位置而不需重读整个对话历史的机制。

在 OpenTalon 中，这些工具是 Claude Code 构建 OpenTalon 本身的工具，也是 OpenTalon 通过 CLI 向用户暴露的同一工具集。第 9 章实现 CLI 智能体循环时，Read、Edit 和 Bash 将成为 OpenTalon 工具集的核心。我们不仅在抽象学习这些工具，我们在学习它们因我们将实现它们。
