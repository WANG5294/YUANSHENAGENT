## 章节 3.5：第一个智能体任务：绘制 OpenTalon 架构

第一个任务非生产性，而是诊断性。其目的非产生输出——而是让你在项目上首次观察感知-推理-行动循环的执行。

在 `opentaion/` 目录打开 Claude Code。给它精确提示：

```
Do NOT write any code or create any files. Read the project structure
and the CLAUDE.md file, then produce a brief description of what this
project is, what its three components are, and what constraints govern
how it will be built. Ask me if anything is unclear about the project
before writing your description.
```

"不要编写任何代码或创建任何文件"这条指令有分量。无它，Claude Code 可能将"生成描述"理解为开始构建的邀请。明确约束将任务保持在探索（Explore）阶段——探索 → 计划 → 编码 → 提交工作流内。我们还没在构建，而是在确认 Claude Code 理解了我们即将构建什么。

### Claude Code 做了什么

观察工具调用。Claude Code 会用 Glob 列出目录结构，用 Read 加载 CLAUDE.md，然后推理找到的内容：

```
$ claude
> Do NOT write any code or create any files. Read the project structure
  and the CLAUDE.md file...

● Glob: opentaion/**/*
  → cli/, web/, api/ (空目录), CLAUDE.md
● Read: CLAUDE.md
  → 加载了 10 个部分，200+ 行
● 思考中...
```

读取后，Claude Code 问一两个澄清问题——或如果 CLAUDE.md 足够清晰，直接进入描述。典型输出：

```
这是 OpenTalon，一个带有 Web 平台的 macOS CLI 编码智能体，
用于用户注册和 Token 用量追踪。

三个组件：
- CLI：Python 智能体（Click + Rich），通过 Homebrew 分发
- API：FastAPI 用量代理（代理到 OpenRouter，计量 Token）
- Web：Vite + React 仪表盘（注册、API 密钥、用量图表）

来自 CLAUDE.md 的约束：
- 技术栈是固定的——不要建议替代方案
- CLI 包管理器：uv（不是 pip）
- 目前还没有代码——下一个里程碑是第 2.5 节

准备好开始时告诉我。
```

此描述可能与上面示例不同——确切措辞取决于 Claude Code 如何解释 CLAUDE.md 及它问什么问题。差异预期且不重要。重要的是 Claude Code 已经：

1. 从磁盘读了实际 CLAUDE.md（而非总结其描述）
2. 用 Glob 列出实际目录结构（而非假设）
3. 产生基于所找到内容的总结，而非基于猜测

### 刚刚发生了什么

那就是感知-推理-行动循环在运行。Glob 和 Read 是感知。"思考中..."是推理。描述是行动。然后循环终止：响应中无工具调用，控制权返回你。

此任务未产生代码。未创建任何文件。输出是你也能写的段落。这些都不重要——教训在工具调用的序列而非输出。

教训：Claude Code 现已确定方向。它知道了项目是什么、有哪些约束、从哪开始工作。更重要的是，它知道它读了什么、找到了什么——而非被告知要假设什么。下个打开此项目的会话需重读相同文件，因会话记忆不持久。但文件本身持久。CLAUDE.md 在那，目录结构在那，会话前检查清单确保 Claude Code 总是先读它们。

在 OpenTalon 中，这第一个探索任务是里程碑 2。非因输出有价值，而是因完成它确认了开发环境配置正确：Claude Code 可读项目、理解 CLAUDE.md、用 Glob 和 Read 感知结构、产生接地输出。后续每个会话将以相同方式开始——由文件引导，非由记忆引导。

第一部分完成。你已有思维模型：智能体是什么、上下文是什么、工具是什么。第二部分构建将这些概念投入实践的平台。
