## 章节 2.5：从头编写 OpenTalon CLAUDE.md

理解 CLAUDE.md 最佳方式是阅读一个必须实际工作的文件。

若你下载了本书仓库，你已有它：项目根目录下 `CLAUDE.md` 是 Claude Code 用来编写你读到每个章节的实际配置文件。非模板或教学示例——是生产基础设施。本节逐段介绍其十部分，解释每个结构性决策，并以开启 OpenTalon 的行动结束：初始化项目目录并首次打开 Claude Code。

### 写作前检查清单（第 3 节）

检查清单是首需理解之物，因它以可操作形式表达本书核心问题。Claude Code 在会话间无记忆，检查清单即替代品。写任何章节前，Claude Code 须读 `progress.md`（我们停在哪？）、目标章节大纲条目（本节完成什么？）、前一章节文件（上节说了什么？）和 `opentaion-state.md`（现存在什么代码？）。

检查清单每项存在，皆因缺失导致过真实问题。大纲条目防止范围漂移。前一章节防止突兀语调断档。状态文件防止 Claude Code 写关于不存在代码的内容。检查清单非形式主义，而是最小可行记忆替代品。

你写自己 CLAUDE.md 时，版本会不同——但你需要一个。问题是：Claude Code 每会话开始需读什么才能避免盲目开始？

### 技术栈部分（第 4 节）

技术栈部分用了"固定（fixed）"一词。选择此词是有意且重要的。"固定"意味着：不要辩论这些选择，不要建议替代方案，未明确批准不要用其他库。

此坚定性的原因是架构连贯性。OpenTalon 跨越 21 章和多个会话。若技术栈漂移——某章用 `uv` 而另一章用 `pip`，或某章引用 `shadcn/ui` 而该组件已不在栈中——不一致累积到 110 章节中变得不可维护。"固定"防止模型做局部合理但全局违反本书连贯性的决策。

对你自己的项目，技术栈部分应列出不仅用什么，还不用什么——且对两者都要具体。

### 唯一规则（第 10 节）

CLAUDE.md 最后部分最重要：系统服务于工作，永不反过来。每个配置、每个技能文件、每个钩子——若它造成损害产出的摩擦，就改变它。此规则防止 CLAUDE.md 变成它原本要支持的工作的约束。

这是覆盖所有其他规则的规则。第 21 章将在完整项目上下文中回到此规则。

### 里程碑：初始化 OpenTalon 项目

这是 OpenTalon 软件开始的时刻。创建目录结构，放置 CLAUDE.md，运行 `git init`。从此刻起，书中每个代码示例都是真实的。

```bash
# 创建项目目录
mkdir -p opentaion/{cli,web,api}
cd opentaion
git init
```

```
Initialized empty Git repository in /path/to/opentaion/.git/
```

现将书仓库根目录下 CLAUDE.md 复制到 `opentaion/` 目录，或用上述结构新建。若从头写自己的，从五必需部分开始：项目标识、仓库结构、会话前检查清单、技术栈和唯一规则。随项目发展添加其余。

```bash
# 验证结构
ls -la opentaion/
```

```
drwxr-xr-x  cli/
drwxr-xr-x  web/
drwxr-xr-x  api/
-rw-r--r--  CLAUDE.md
```

从项目根目录打开 Claude Code：

```bash
cd opentaion
claude
```

```
✓ Claude Code v1.x.x
  Project: opentaion
  CLAUDE.md: loaded (root)
  Memory: no previous session
>
```

### 刚刚发生了什么

Claude Code 启动时读了 CLAUDE.md。它现在知道项目标识、技术栈约束和会话前检查清单。还不知代码库——目录空——但知道代码库将按什么规则构建。

此即里程碑 1 的全部目的：非产出代码，而是建立将指导后续每个编码会话的信息环境。CLAUDE.md 非已完成工作的文档，而是未来工作的契约。

在 OpenTalon 中，你刚初始化的 CLAUDE.md 正是 Claude Code 在第 9 章实现 CLI 智能体循环时、第 11 章搭建 Web 平台时、第 19 章部署到 Railway 时将读的同一 CLAUDE.md。每个会话将继承你刚建的约束。对这些约束的更改将在版本控制中被跟踪——CLAUDE.md 的 Git 历史即项目决策日志。

第 3 章将用此已初始化项目演示第一个真正的智能体任务：让 Claude Code 探索空仓库并描述自己的开发环境。
