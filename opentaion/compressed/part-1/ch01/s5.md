## 章节 1.5：创始问题：什么使一个工具具备智能体能力？

此问题值精确回答：工具是智能体的，当它能感知环境、采取行动、观察结果并决定下一步——而无需每一步等待人类时。

此定义中每个词都有分量。*感知*意味着真正的感知——读实际文件、运行实际命令——而非总结环境描述。*采取行动*意味着产生真正改变——写入文件系统、执行代码——而非生成文本后由人类决定是否行动。*观察结果*意味着工具接收来自自身行动的反馈——测试输出、错误消息、变更状态——且此反馈进入其上下文。*无需每一步等待人类*是区分智能体与非常好助手的关键：循环在任务开始和结束间自主运行。

此定义形成一个频谱。自动补全在一端：预测下一 Token，交付，然后等待。它没环境，不行动，不观察任何东西。对话式 AI 助手——多数人听到"AI 编码工具"时所想的——在稍远处：接收问题描述，返回解决方案描述。比自动补全好，仍非智能体。人类仍是反馈循环。

Claude Code 在更远处。它感知真实文件，执行真实命令，将真实工具结果纳入推理。它自主运行感知-推理-行动循环数十次迭代。它在任务完成时终止，而非人类决定停止指令时。据上述定义，Claude Code 是智能体的——非营销主张，而是其所作所为的结构性描述。

完全自主性（full autonomy）在频谱最远端：可被赋予目标并无限追求、处理所有障碍、做所有决策、任何时候不需人类参与的智能体。那不是 Claude Code，不是 OpenTalon，不是今天可靠存在的任何系统。完全自主智能体在约 23% 的复杂真实世界软件任务上成功，其余需干预。理解工具在频谱上的位置，是不期望它们做不能做之事的先决条件。

对 OpenTalon，此定义决定了最小可行设计。CLI 须能感知文件，须能执行命令并读取输出，须在感知与行动间维持自己的循环，不需用户将结果粘贴回提示。低于此标准者皆智能包装器。区分真正智能体的三个能力——持久化工具使用、环境反馈、多轮规划——直接映射到这些需求。

OpenTalon V1 在频谱何处？在 Claude Code 与完全自主性间，更近 Claude Code。它在熟悉代码库上处理明确任务，不会处理模糊需求、新颖架构或需同时理解整个代码库的任务。这是正确起点：足够有能力以有用，足够简单以理解，足够有限以诚实。

创始问题已得到回答。循环是真实的，运行于真实工具。现在还有第二个问题：循环在会话间无记忆，每次启动都是冷启动。第 2 章关于构建循环在跨真实项目长度上保持连贯所需的记忆。
