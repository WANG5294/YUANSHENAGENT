## 章节 P.1：元项目（Meta-Project）

本书正用其所授工具写成。

这不是思想实验或章节收尾。你读到的书稿，用 Claude Code——我们将花 21 章学习的同一智能编码助手——起草、修改并组织。每节产生这些文字的会话，都遵循本书所述系统：指示智能体行为的 `CLAUDE.md`、编码流程的技能文件、跨会话延续的进度跟踪。系统运行于自身之上。此即元项目。

核心主张：智能体工程是一门拥有可学习原则的学科——非提示技巧集，亦非待工具升级的问题。主张易提，证明难。证明即我们用于构建本书及其内部软件的系统。

这套软件即 OpenTalon：可工作的 macOS CLI 编码智能体，附用户注册与 Token 用量仪表盘的 Web 平台。至第 21 章末，OpenTalon 可通过 Homebrew 安装，部署于 Railway 和 Vercel，有邮箱即可用。非玩具，非伪代码。它能编译、运行、发布——构建它的每项技术，均在首次应用章节中解释。

为何同时做两事？描述智能体工程却不付诸实践的书，往好说缺乏说服力，往坏说就是误导。第三部分的每个模式，皆因构建 OpenTalon 时所需。第二部分的每项配置，皆因我们在本书开发环境中使用。系统服务于工作，工作验证系统。若此循环耳熟——这正是 Claude Code 编写软件的同一反馈循环，亦是全书论证核心。

在 OpenTalon 中，元结构有具体体现：仓库同时包含书稿与构建中的软件，Claude Code 同时编写两者。第 5 章将读到的 `.claude/skills/` 目录，正是产出这些章节的同一目录。跟踪章节完成的 `book/progress.md`，正是第 2 章作为智能体遗忘解决方案所授的同一机制。你可边读系统，边读关于系统的描述。非巧合——此乃论点物理化。

本书写给独立开发者：独自发布产品者，经历过 AI 代码能编译但做错事者，希望从内部理解智能体系统而非看营销页面者。无需事先用过 Claude Code。需能舒适阅读 Python 和 TypeScript，能在终端运行命令，能容忍构建完整答案过程中暂时的不完整。

OpenTalon V1 会有粗糙边缘。非免责声明——这正是重点。独立开发者的首个智能体系统本应有粗糙边缘。你从中习得的东西，才是本书真正要教的。
