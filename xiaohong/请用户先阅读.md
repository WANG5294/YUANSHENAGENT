# 请用户先阅读

## 这个 Agent 是什么（功能）

小红 v3 是面向 ESP32 开发板的单片机开发 Agent：你用一句话描述目标，它自动完成"编写 MicroPython 代码 → 烧录为板上 main.py（原程序自动备份）→ 实机测试（串口回读 + 麦克风闭环听声）→ 汇报结果"的完整主线，全程每轮输出循环摘要，任务结束自动归档流程日志并提取可复用经验（经你确认才保存）。

## 怎么用（用法）

1. 在项目根目录运行 `python3 xiaohong/xiaohong_v3.py`；
2. 直接输入**目标**即可，如"按key1播放do音"、"让绿灯闪三下"——不用写"请编写/烧录/运行"，这些步骤已内置在 system prompt 的主线里；
3. 运行中每轮显示一行循环摘要和工具调用情况（`[MCP]`/`[本地]` 标注）；结束后看到"⏳ 正在保存记录"时**不要关程序**，等"✅ 记录已保存"再继续；
4. 若提取到候选经验，会展示全文请你按 `y/n` 确认；选 n 还可保留上下文继续对话修改；
5. 常用命令：`/model` 切换大模型（kimi-k3 与 deepseek-v4-pro 可换着用）、`/work` 探测环境、`/wiring` 查看接线、`/skill` `/tool` 查看能力、`exit` 退出；
6. 每个任务的产出、日志、报告都在 `xiaohong/files/<时间戳_任务名>/`；板上被覆盖的原 main.py 在 `backups/`。

## 如何添加 MCP 和 Skill

**加 Skill**：把新硬件说明写成带 frontmatter 的 md（`---` 包住 `name:`（小写短横线）与 `description:`（写清什么任务该加载），下接正文），运行中输入 `/doc 该文件路径` 即导入生效，无需改代码。
**加 MCP**：在 `xiaohong_v3.py` 的 `MCP_SERVERS` 列表里加一行 `{"name": "服务器名", "cmd": [解释器, 服务器脚本路径]}`，服务器需支持 stdio JSON-RPC（用 FastMCP 写函数即工具，docstring 即说明）。重启后其工具自动注册。
