# RMT 音频"哒哒音"问题分析与无缝播放方案

日期：2026-07-16
涉及文件：`sine440_fs7920.py`（有缺陷的旧版）→ `sine440_loop.py`（修复版，已烧录为 main.py）
硬件：ESP32-D0WD，GPIO25 → J11 → Q3(NPN 8050) → 扬声器；固件 MicroPython v1.18

## 1. 现象

播放 3 秒 440Hz 正弦（采样率 7920.8Hz，18 点/周期）时，全程听到
**14 次均匀分布的"哒哒"声**，间隔约 250ms（每秒 4 次）。

## 2. 原因分析

旧版把周期波形铺成 **1980 采样（≈250ms）的大块**，用 13 次
`rmt.write_pulses(dl, True)` 依次发送：

```
reps = int(3.0 × 7920.8) // 1980 + 1 = 13 块
```

**2026-07-16 板上实测修正**：块间空隙恒约 **35µs**，与块大小无关
（88/176/352 采样块均为 34~35µs）——说明 v1.18 的列表转换与上一块播放
是**重叠**的，空隙只剩驱动"停止→重启"的固定延迟，而非最初推测的
"几毫秒转换期闲置"。

但 35µs 照样可闻：PWM-DAC 的"静音"不是低电平，而是约 50% 的直流偏置。
若切口落在占空比 ≈50% 的相位上，输出突然多出一段 35µs 的低电平，
相当于往偏置电流里打了一个**满摆幅窄脉冲**——单发是轻微的"嗒"，
每 250ms 一发的固定节奏就成了清晰的哒哒声。

账目核对（与听感完全吻合）：

| 事件 | 次数 |
|---|---|
| 开头 0→偏置 突跳（此版未加斜坡） | 1 |
| 12 个块间接缝（低电平空隙） | 12 |
| 结尾 偏置→0 突落 | 1 |
| **合计** | **14** |

补充：更早的 20kHz 版（`sine440_pure.py`，125ms 一块、24 块）存在同样的接缝，
只是首尾加了斜坡、接缝更密（8 次/秒）更易被主音掩蔽，听感上表现为"粗糙"而非
清晰的哒声。**分块 write_pulses 的接缝在持续纯音下必然可闻**，此前记录的
"块间衔接毫秒级"是达标错觉——毫秒级的直流跌落恰恰就是爆音。

## 3. 修复方案：RMT 硬件循环

RMT 每个通道自带 64 条脉冲的硬件内存。一个周期 18 采样 = **36 条脉冲，
放得进去**，于是让硬件自己循环，CPU 完全退出喂数：

```python
rmt.write_pulses(up, True)      # 10ms 直流斜坡 0→CENTER，防开头爆音
rmt.loop(True)
rmt.write_pulses(period, True)  # 36 条写入硬件内存，自循环，立即返回
time.sleep(DUR)                 # 播多久睡多久，零 CPU、零拷贝、零接缝
rmt.loop(False)                 # v1.18 语义：当前圈播完再停
rmt.write_pulses(down, True)    # 10ms 斜坡 CENTER→0，防结尾爆音
rmt.write_pulses((1,), False)   # 留在低电平，Q3 截止
```

斜坡与循环体之间仍有一次接缝，但小列表（36/158 条）的转换只需几十微秒，
远短于可闻阈值，实测无声。

## 4. 教训与规则（以后写 RMT/PWM-DAC 播放代码必须遵守）

1. **持续音绝不允许输出中途闲置**。分块接缝空隙实测恒 35µs（与块长无关），
   落在占空比 50% 相位上就是一个满摆幅窄脉冲，节奏性出现即为哒哒声。
2. **周期波形（单音、试音、校准音）一律用 `loop(True)` 硬件循环**，
   条件：一个周期的脉冲条数 ≤ 64。这同时是最省 CPU、频率最准的做法。
3. **非周期波形（包络音、歌曲）分块流水时，必须做"电荷补偿"**。
   "把切口藏进波形低电平段"已被实测证伪（2026-07-16：21 声哒、包络衰减后
   反而更响）——哒声的本质是 35µs 空隙**丢失的平均电平**（电荷亏空
   ≈ 35µs × 偏置），深度与信号音量无关，缓边沿补不回来；信号越衰减
   掩蔽越弱，哒声越露。正确做法：**每块最后一个采样的高电平加宽
   35µs 对应的 tick 数**（本例 233 tick，保持采样总周期不变），把亏空
   提前补上，凹坑与补偿脉冲相邻一两个采样内，经膜片低通后相抵。
   范例 `key440_note.py`。转换与播放重叠，块内 fill/gc 只要短于块播放
   时长就不产生额外空隙。
4. **首尾直流斜坡（约 10ms，0↔CENTER）永远要加**，与 ADSR 包络无关，
   消的是"载波突然出现/消失"本身。
5. 听到**均匀节奏的杂音**，第一反应是数频率：杂音次数/秒 ≈ 某个分块或
   定时器的节拍 → 直接定位到代码里对应的块长参数。本次 4 次/秒 ↔ 250ms 块。

## 5. 相关参数速查

- RMT tick = 80MHz / clock_div；每采样 TICKS 个 tick；采样率 = 80MHz/(div×TICKS)
- 本例：div=101, TICKS=100 → 7920.8Hz；18 点/周期 → 440.04Hz（差 0.2 音分）
- 占空比钳位 1..TICKS-1（时长 0 = RMT 序列结束标记）
- 中心 CENTER ≈ 50% TICKS；振幅按 20kHz 版参数等比缩放（62/125 → 50/100）
