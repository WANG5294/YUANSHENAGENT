# 麦克风闭环调试（让 agent"听"到实机声音）

## 为什么需要

主机预览 WAV 只验证**算法**；实机声音还叠加了 Q3 开关、喇叭机械特性、
供电纹波、RMT 块间空隙、GC 停顿。哒哒音就是只在实机上才暴露的典型。
闭环 = 麦克风录下喇叭真实输出 → 与预览对比 → 差异即"实机才有的问题"。

## 录音通道现状（2026-07 实测）

- 本机是 VirtualBox 虚拟机，声卡为模拟 Intel ICH（含 MIC ADC），
  走 PipeWire，`arecord -D default` 可用。
- **实测录到全零** → 必须先做两件事：
  1. VirtualBox 菜单：**设备 → 音频 → 勾选"音频输入"**
  2. 宿主机确认麦克风未静音、有权限（`pavucontrol` 输入页看电平表）
- 每次闭环调试**先跑 `mic_check`**，确认非全零并记下噪声底 RMS。
- 麦克风尽量凑近喇叭（几厘米），提高信噪比、减少房间混响。

## 触发方式：物理按键 agent 按不到

KEY1 是实体按键。闭环测试要求播放可**软件触发**：

- 把播放逻辑写成模块函数（`piano.py` 定义 `play()`），主程序
  `import piano` 后在按键循环里调用——人按键、agent 走 REPL 两不误。
- agent 侧：`play_and_record("import piano; piano.play()", "rec.wav", 4)`
  ——先启动 arecord，0.5s 后经 `mpremote exec` 触发，录完自动检查峰值。

## 对比时什么可信、什么不可信

| 特征 | 可信度 | 说明 |
|---|---|---|
| 基频（音分偏差） | 高 | 频率不受通道染色影响，±10 音分内算准 |
| 哒声有无/周期 | 高 | 录音里等间隔尖峰且预览没有 → 实机接缝问题；间隔≈CHUNK/RATE 可直接定位到分块参数 |
| 包络形态（相关系数） | 中 | 起音/衰减趋势可比，绝对响度不可比 |
| 谐波比例/音色 | 低 | 蜂鸣器/小喇叭+虚拟机麦克风严重染色，只看趋势不看绝对值 |

## 标准闭环流程

1. `mic_check` —— 非全零、记噪声底
2. `analyze_wav xxx_preview.wav 期望f0` —— 先确认参考本身干净
3. `play_and_record "import piano; piano.play()" rec.wav 4`
4. `analyze_wav rec.wav 期望f0` —— 看基频、哒声报告
5. `compare_audio rec.wav xxx_preview.wav` —— 互相关对齐后比包络/瞬态
6. 判读：
   - 录音有规律尖峰、预览没有 → 块接缝，回 SKILL.md 第 4 条
   - 首尾各一个孤立尖峰 → 缺直流斜坡（爆音）
   - 基频偏差大 → 检查 clock_div/TICKS 推算的实际采样率
   - 包络相关低但都无尖峰 → 可能录到环境噪声，靠近喇叭重录
